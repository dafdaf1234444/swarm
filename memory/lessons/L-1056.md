# L-1056: eval C1 baseline staleness — hardcoded artifact path silently loads stale data
Session: S443 | Date: 2026-03-02 | Domain: evaluation | Level: L3
Confidence: MEASURED (n=2 baselines, s428→s429, c1 rate 6.6%→3.8%)
Cites: L-601, P-283, F-EVAL1, L-988

## Finding
eval_sufficiency_data.py hardcoded `f-con1-baseline-s428.json`. A newer baseline
(s429, c1=3.8%) existed but was silently ignored. Two bugs compounded: (1) hardcoded
path, (2) schema mismatch — s429 nests rate under c1_duplicate_work.rate_lane_level
while s428 had it at top level, so con1.get("c1_rate_lane_level", 0.066) fell through
to the stale default even if the file was loaded.

## Fix
Dynamic glob picks latest: `sorted(conflict_dir.glob("f-con1-baseline-s*.json"))[-1]`.
Normalization: if top-level key missing, check nested path. `_source_file` key added for
provenance. c1_source string in score_collaborate uses `con1.get("_source_file")`.

## Impact
c1_rate: 6.6% → 3.8% (correct s429 value). Now below 5% threshold for discrete score=2.
Continuous score +~0.1. Source file accurately reported in output.

## Rule
Experiment artifact accessors must use dynamic glob (latest-wins). When loading data
that may change schema across sessions, normalize to canonical keys immediately after
loading — never let schema drift silently degrade measurement accuracy.
