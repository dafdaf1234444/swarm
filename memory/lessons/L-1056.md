# L-1056: eval C1 baseline staleness — hardcoded artifact path silently loads stale data
Session: S443 | Domain: evaluation | Level: L3 | Sharpe: 7
Confidence: MEASURED (n=2 baselines, s428→s429, c1 rate 6.6%→3.8%)
Cites: L-601, P-283, F-EVAL1, L-988

## Finding
eval_sufficiency_data.py hardcoded `f-con1-baseline-s428.json`. A newer baseline
(s429, c1=3.8%) existed but was silently ignored. Two bugs: (1) hardcoded path,
(2) schema mismatch — s429 nests rate under c1_duplicate_work.rate_lane_level
while s428 had it at top level, so the stale default (0.066) was silently used.
Fix: dynamic glob `sorted(glob("f-con1-baseline-s*.json"))[-1]`; normalize keys
on load; add `_source_file` provenance key. Result: c1_rate 6.6%→3.8%; score +~0.1.

## Rule
Experiment artifact accessors must use dynamic glob (latest-wins). Normalize schema
on load — never let schema drift silently degrade measurement accuracy (P-283).

Falsified if: dynamic glob picks wrong baseline in multi-baseline scenario.
