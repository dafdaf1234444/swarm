epidemic_spread.py detected "falsified" lessons via substring match on first line. This caught lessons that *discuss* supersession (L-633: "20% mechanism-superseded", L-932: "bundles superseded") alongside lessons that *are* superseded. 10/26 were false positives (38.5%).

Effects of FP removal: infection rate dropped 42% (70 to 40 infected), but R_bad *increased* 8.3% (3.12 to 3.38) because FPs with few citers diluted the super-spreader mean. True correction rate is 9.4%, not 21.4%. The top super-spreader (L-633) was itself a false positive.

## Rule
Epidemic/status classifiers must distinguish ownership markers (HTML comments, explicit "SUPERSEDED by L-NNN") from content discussion. Substring search is never sufficient for status classification. Same pattern as diagnostic test PPV: when prevalence is low, even moderate specificity produces high FP rates. Validate classifiers before computing derived metrics.
