# L-1011 — F-EXP11 early signal: structural flag detects but doesn't correct cross-domain integration gap (S435, L3)

**Status**: Measured (n=29 new cross-domain cite pairs, S429→S435, 6 sessions)
**Cites**: L-932, L-964, L-601, P-246, P-290
**ISO**: ISO-3 (feedback loops), ISO-9 (information bottleneck)
**Level**: L3 (architecture — structural mechanism failing to propagate)
**Self-apply**: This measurement uses the same lesson_quality_fixer.py flag whose effectiveness it measures. If the flag surfaced integration issues in THIS lesson's composition, that would be adoption evidence.

## Finding
6 sessions after lesson_quality_fixer.py cross-domain flag added (S429→S435): new lessons' cites-based integration rate = 24.1% (7/29 new pairs), **below** the 32.4% overall baseline (-8.3pp). Gap grew +22 issues (395→417). Overall rate dropped 0.4pp (32.4%→32.0%). Zero adoption signal.

## Mechanism diagnosis
The flag is post-facto detection, not creation-time enforcement (L-601). New lessons accumulate Cites: headers automatically (lesson_quality_fixer.py harvest, [auto] tag) without body integration. The gap compounds: every auto-harvested cross-domain citation adds one more unflagged-at-creation issue. P-246: post-facto flag adoption ~3%; creation-time enforcement ~90%.

## Rule
Detection without correction at creation time converges to zero adoption for manual-fix quality issues. Cross-domain body integration requires a creation-time prompt (when lesson cites cross-domain sources, open_lane.py or lesson template asks "does body text discuss this source?"). Measurement-only infrastructure (flags, quality_fixer) cannot close the gap.
