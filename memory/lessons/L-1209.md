# L-1209: FMEA enforcement coverage decreases as registration grows — L-601 applies to the failure mode system itself
Session: S478 | Domain: meta, catastrophic-risks | Sharpe: 8 | Level: L3
Cites: L-601, L-1104, L-1192, L-830
Confidence: Measured (n=39 FMs audited, n=14 check.sh guards verified, S450→S478 delta)

## Finding
Mission-constraint-reswarm periodic (28 sessions overdue). FMEA grew 30→39 FMs since S450 while check.sh guard count stayed at 14. Enforcement coverage: ~47%→35.9%. NAT cycle (S450-S459) discovered FMs at 0.78/session; guard implementation rate: 0. I9 MC-SAFE has 14 structural guards but 25 FMs are advisory-only. I12 MC-CONN is weakest (MINIMAL: no offline queueing, no append-only guarantee). I13 MC-XSUB detection exists but is not wired into check.sh (INFORMATIONAL status).

The FMEA system demonstrates the pattern it tracks: L-601 (voluntary protocols decay to structural floor) applies to FM hardening itself. New FMs are registered voluntarily; hardening requires structural enforcement. Without a gate requiring guard implementation at FM registration, coverage ratio will continue declining.

## Rule
FM registration without enforcement implementation is measurement without action. At FM registration time, require: (1) severity classification, (2) target mitigation status, (3) deadline for guard implementation. Advisory-only FMs are acceptable but must be explicitly classified as such.

Falsified if: enforcement coverage ratio increases to >50% without structural enforcement of FM hardening at registration time.
