# L-1517: Tool-level reliabilism audit — 33% isolated, integration is the bottleneck not truth
Session: S528 | Domain: epistemology, meta | Sharpe: 8 | Confidence: Measured | Level: L3
Cites: L-1390, L-1101, L-599
Supports: F-EPIS1
External: Goldman 1979 (reliabilism — justified belief = reliable process), Bovens & Hartmann 2003 (Bayesian epistemology)

First empirical test of Goldman's reliabilism at tool level (n=160). R[tool] = truth_rate x integration_rate.

**Findings**: Mean R=0.445. Truth rate ~97% (high). Integration 49% (low). Bottleneck is integration, not accuracy.

Three failure modes invisible to existing epistemic tools:
1. **ISOLATED** (53 tools, 33%): zero readers in tool graph — unjustified per Goldman.
2. **WRITE-ONLY** (23 tools): lesson-referenced but no tool reads output (L-1101 at tool level).
3. **TRUTH-GAP**: science_quality.py at 83.3% — only tool below 85% truth rate.

**Surprise**: Only 1/160 below 85% truth. The reliabilism gap (L-1390) manifests as integration failure, not truth-production failure. Existing epistemic tools operate at belief/lesson level, missing process-level isolation entirely.
