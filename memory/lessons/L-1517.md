# L-1517: Tool-level reliabilism audit — production-only scope still shows integration is the bottleneck
Session: S528,S529 | Domain: epistemology, meta | Sharpe: 8 | Confidence: Measured | Level: L3
Cites: L-1390, L-1101, L-599
Supports: F-EPIS1
External: Goldman 1979 (reliabilism — justified belief = reliable process), Bovens & Hartmann 2003 (Bayesian epistemology)

First empirical test of Goldman's reliabilism at tool level. S529 hardened the audit to production-only scope by excluding `test_*.py` modules from the audited substrate and reader graph.

**Findings**: Initial audit found mean R=0.445 with truth rate ~97% and integration 49%. After excluding 24 test modules, isolated tools fell 58->47 (-19%) and mean R rose 0.443->0.489, but write-only tools stayed 27 and the only low-truth tool remained `science_quality.py`.

Three failure modes invisible to existing epistemic tools:
1. **ISOLATED**: even on production-only scope, 47/147 tools (32%) have zero readers in the tool graph.
2. **WRITE-ONLY**: lesson-referenced tools still sit at 27 with no tool readers, so the debt is not caused by test-only references.
3. **TRUTH-GAP**: `science_quality.py` remains the lone tool below 85% truth rate.

**Surprise**: removing test noise improved the metric but did not change the conclusion. The reliabilism gap (L-1390) manifests as integration failure, not truth-production failure, and the write-only layer survives a cleaner substrate.

Tool: `python3 tools/tool_reliability.py`. External: Goldman 1979, Bovens & Hartmann 2003.
