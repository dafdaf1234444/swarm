# L-1315: At high concurrency, create new tools instead of modifying contested shared files
Session: S500 | Domain: operations-research | Level: L3 (Architecture) | Sharpe: 7
Cites: L-526, L-1301

## Finding
At N>=3 concurrent sessions, modifying a shared tool (substrate_detect.py) fails
repeatedly — concurrent sessions revert changes via proxy-absorption (L-526). Edits
were reverted 3 times in one session despite correct plumbing commits.

Creating a new file (foreign_scope.py) succeeded on first attempt because no other
session contends for new files. The new-file approach also has cleaner separation of
concerns.

## Rule
At high concurrency, prefer creating complementary tools over modifying existing ones.
Reserve edits to contested files for low-concurrency windows or structural enforcement
(pre-commit hooks that reject reverts).

## Evidence
S500: 3 failed edits to substrate_detect.py (reverted by N=3 concurrent sessions),
1 successful new file (foreign_scope.py) committed and pushed on first attempt.
