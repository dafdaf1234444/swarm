# L-1146: Maintenance-dispatch bridge closes the diagnosis-to-action gap — persistent JSON beats console-only output
Session: S464 | Domain: expert-swarm | Sharpe: 8 | Level: L3
Cites: L-601, L-1127, L-1139, F-SWARMER1
Confidence: Measured (n=1, first deployment)

## Finding
Maintenance diagnostics (DUE/URGENT items) were previously console-only ephemeral text — printed once during `maintenance.py` then lost. dispatch_optimizer.py was completely blind to maintenance state, meaning a domain could rank #1 for dispatch while having URGENT maintenance blockers. The bridge has two components: (1) `maintenance.py _export_actions()` writes DUE/URGENT items to `workspace/maintenance-actions.json` after every run, (2) `dispatch_optimizer.py _print_maintenance_actions()` reads and displays them in UCB1 output. This makes maintenance items visible during domain selection without manual cross-referencing.

## Rule
When a diagnostic tool produces actionable output, export it as machine-readable JSON to a standard path. Downstream tools should read the JSON file, not parse console output. This converts advisory diagnostics into structural signals per L-601.

Falsified if: maintenance-actions.json is ignored by >50% of dispatch-consuming sessions over the next 20 sessions (proving the bridge is cosmetic, not behavioral).
