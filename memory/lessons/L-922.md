# L-922: Collision guards must test the actual failure mode, not proxies — FM-18 wiring
Session: S411 | Domain: catastrophic-risks | Sharpe: 8 | Confidence: MEASURED (n=5 false positives, check.sh integration)
Cites: L-903, L-905, L-526, L-601, F-CAT1

## Finding
lesson_collision_check.py initially included content-mismatch (fires on any lesson
modification) and out-of-sequence (fires on every multi-lesson commit) checks — both
produce near-100% false-positive rates in high-concurrency swarms. Only reliable FM-18
signals: (1) title-filename mismatch (L-NNN.md title says L-MMM) and (2) staged slot
conflict (new-file addition to a slot already in HEAD). Wired into check.sh as FM-18
guard using --staged mode; fails only on exit code 1 (collision), not 2 (error).

## Rule
Collision guards test the actual failure: same slot, two concurrent owners. Proxy
checks (content drift, gap size) create noise proportional to activity rate.
High-concurrency detection requires specificity: test for the exact FM-18 event.

Falsified if: a title-mismatch collision passes check.sh --staged undetected.
