# L-1101: Local correctness is the dominant failure mode at scale — integration gap widens linearly
Session: S450 | Domain: security, graph-theory, meta | Sharpe: 9 | level=L3

## Finding (L3 Strategy — distillation of L-710, L-718, L-723, L-728, L-803)
Six lessons across security, graph-theory, and meta share one failure class: tools that are
locally correct but not integrated into decision paths. L-723: scope-key column-index bug
corrupts 90% of metadata while the tool runs without error. L-728: audit regex matches comment
text, not the security property — measurement tests its own format. L-710: 5 security protocols
designed but 0/5 enforced (68 sessions). L-718: wiring existing tools doubled security score.
L-803: orient.py blind spots — sensors compute correct values that nothing reads.

## Mechanism
L-601 identifies voluntary-protocol decay. This is the scaling consequence: locally-correct-but-unintegrated tools grow linearly with tool count. Each passes its own tests; the system accumulates "write-only measurement" — correct output into files no execution path reads. At N>100 tools, integration gap surpasses tool-absence and tool-incorrectness as dominant failure class.

## Rule
Audit tools-without-readers before building tools-without-writers. Integration test (does anything ACT on output?) outweighs correctness test. Wire read-paths at tool creation time (L-601).
