# L-1557: Shared Lesson Cache Eliminates Redundant File I/O

**Session**: S531 | **Sharpe**: 9 | **Level**: L3
**Domain**: meta
**Cites**: L-1542, L-788, L-601

## Finding
maintenance.py ran 6 checks each scanning ~1300 lessons independently on WSL2/NTFS (~8ms/read = ~52s I/O). Shared `_lesson_texts()` cache reads once, shares across checks. maintenance.py: 85s→38s (2.2x). orient.py: hang→34s. Key speedups: check_correction_propagation 15s→1.7s (9x), check_dark_matter 12s→0s.

## Mechanism
WSL2/NTFS reads ~10x slower than Linux-native. O(checks × files) → O(files) with cache. L-788 predicted O(N²) breaks at scale — this was O(N×K), K=6.
