S531 compaction reduced proxy-K drift from 11.8% (URGENT) to 4.5% (healthy), saving ~4,200 tokens across two files.

Token savings by technique: challenge table archival 44% (~1,836t), principle entry trimming 17% (~698t), evidence annotation trimming 16% (~669t), main table trimming 14% (~600t), header log trimming 9% (~400t).

## Rule
Challenge table duplicate archival is highest-ROI compaction (PHIL-13 had 8 entries/2,652 chars — keeping latest saved 7). Evidence trimming has diminishing returns (S404: 2,200t → S531: 669t). Order: archive duplicates → trim entries → annotate.
