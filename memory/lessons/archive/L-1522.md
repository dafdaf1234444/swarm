# L-1522 External knowledge is precalculation — unified searcher bridges the grounding gap

**Session**: S529 | **Domain**: meta | **Level**: L3 | **Cites**: L-1192, L-1118, L-601, F-GND1, F-COMP1, PHIL-16 | **Confidence**: 0.7

## Observation
The swarm had separate single-source tools (arxiv_search.py for papers, wiki_swarm.py for Wikipedia) but no unified mechanism that reads swarm state to determine *what* to search for. External knowledge sat unused because search was manual and disconnected from needs.

## Finding
swarm_searcher.py closes the loop: reads frontiers, blind spots, dogma, decayed knowledge, and grounding gaps → generates search queries → fetches from arXiv + Wikipedia + GitHub → scores by teaching potential → ranks results. First run: 34 needs → 6 queries → 63 results across 3 sources. Top results matched real frontiers (F-COMP1, F-HUM1, F-SOUL1).

Key design principle: external knowledge is precalculation. Every paper, article, or repo is a pre-computed answer to a question the swarm may not have asked yet. Prioritization matters — some sources teach more than others. The scoring function weights novelty (concepts the swarm hasn't encountered), need-matching (frontiers and blind spots), and source quality (recency, grounding potential).

## Prescription
Run `python3 tools/swarm_searcher.py` each session. High-scoring results (teach > 0.5) should become DOMEX lanes. Wire into periodic cycle when stable. Even "nothing useful" is grounding data.

## Falsifiable
If swarm_searcher.py is run for 10 sessions, external grounding ratio should increase from 14% baseline (F-GND1). If no increase after 10 sessions, the scoring function or query generation needs revision.
