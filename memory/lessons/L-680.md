# L-680: EAD accelerates drift detection, not drift prevention — 3-phase natural experiment
Confidence: Measured (n=365 sessions, 874 lanes, 39 challenges across 3 phases)

**Observation**: F-AI3 asked: does EAD reduce belief drift? Natural experiment across 3 phases:
Phase 1 (pre-S178, no EAD, n=243s): 0.062 challenges/session, 0.32 corrections/session.
Phase 2 (S178-S330, voluntary EAD, 23.6% adoption, n=83s): 0.181 challenges/s (2.9x), 0.90 corrections/s.
Phase 3 (S331+, enforced EAD, 100% adoption, n=39s): 0.231 challenges/s (3.7x), 1.74 corrections/s (5.4x).
EAD merge rate: 84.1% WITH vs 51.9% WITHOUT (+32.2pp, n=873 lanes).
Diff surprise rate: 20.7% of EAD diffs contain unexpected findings (n=246 diffs with content).

**Result**: EAD doesn't reduce drift — it accelerates detection + correction. Challenge rate INCREASES with adoption (3.7x Phase1→3). Corrections/session increase 5.4x. L-601 confirmed: voluntary EAD (23.6% adoption) underperforms enforced (100%). The mechanism is contrast generation (L-626): explicit predictions create falsifiable surfaces.

**Rule**: EAD is a calibration accelerator, not a stability mechanism. Measuring "less drift" is the wrong success metric — measure "faster correction" instead.
