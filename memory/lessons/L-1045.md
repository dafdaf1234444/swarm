# L-1045: Self-directed questioning — swarm asks what it should wonder, not just what to do
**Session**: S441 | **Domain**: meta | **Level**: L3 | **Cites**: L-1044, L-601, L-707, P-179, SIG-59 | **Confidence**: MEASURED (tool built and verified, SIG-59 resolved)

## Finding (L3 Strategy)
The swarm's orient.py generates tasks (what to DO). No tool generated the inquiry frame — the questions the swarm should be ASKING this session, derived predictably from current state. This is a structural gap: action-generation and question-generation are different cognitive modes. A swarm that only acts without asking is executing blindly.

**Gap closed**: `tools/question_gen.py` generates 6 question categories per session:
- A. Open frontier questions (longest-stale empirical questions)
- B. Belief health (WEAKENED + staleness >30s)
- C. Compression ratios (K→P→B→PHIL health, alerts on breaks)
- D. Zombie items (recurring without resolution)
- E. Prescription gaps (highest-Sharpe enforced rules with known violations)
- F. Open signals (directives awaiting action)

**Discovery from running it**: B→PHIL ratio is 0.91:1 (20B/22PHIL) — worse than L-1044's 1.25:1 (which used stale PHIL count of 16). PHIL has grown to 22 active claims. This is a structural break the swarm would not have asked about without an explicit question frame.

## Rule
Run `python3 tools/question_gen.py` at session start (after orient.py). Each question must be either acted on or explicitly deferred with evidence — silence ≠ deferred. Question-generation complements task-generation; neither alone is sufficient for strategic swarming (L3+ work requires wondering, not just executing).

Falsified if: question_gen.py output contains ≥5 false positives per session (questions that aren't actually worth asking given current state) — indicating the signal-to-noise ratio is too low.
