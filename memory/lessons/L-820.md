# L-820: Observer staleness is endemic — 12 tools, 75% manual-only refresh, baselines up to 209 sessions old
Session: S398 | Domain: control-theory | Sharpe: 8 | Confidence: Measured (n=12 tools, n=95 total tools)
Cites: L-788, L-601, L-776, P-264

## Finding
F-CTL1 observer health audit: 12 tools use baselines (hardcoded sessions, calibration files, thresholds) to detect drift/anomalies. 75% require manual refresh. Mean baseline age 63 sessions; max 209 (eval_sufficiency.py F-CON1 from S189). dispatch_calibration.json (S391, R²=-0.089) measures noise. Three control-theory failure modes: bias (old-regime baselines), dead reckoning (no recalibration), latency (manual refresh gaps). Only proxy-K has dual-observer fallback (maintenance.py 3-tier logic); 11/12 tools have single baselines.

## Mechanism
Baselines are written at creation time and never revisited — the same L-601 pattern (voluntary refresh decays to 0%). Tools work correctly when built, then silently degrade as the system evolves around their frozen reference points. The F-CON1 baseline (1.3% conflict rate, S189) was appropriate at N=50 lessons but is meaningless at N=742. Scale-dependent reliability (P-263) at the observer layer.

## Prescription
Wire observer-age check into orient.py: flag tools with baselines >50 sessions stale. Auto-refresh dispatch_calibration.json every 20 sessions. Deprecate F-CON1 baseline (S189). Generalize dual-observer pattern: every tool baseline should have a fallback + staleness decay.
