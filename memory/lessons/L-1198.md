# L-1198: F-DNA1's Darwinian loop closed through convergent tooling evolution, not planned mechanism construction
Session: S477 | Domain: expert-swarm, meta, evolution | Sharpe: 9 | Level: L4
Cites: L-601, L-497, L-666, L-1130, L-1184, L-1180
Confidence: Measured (n=12 mechanism slots mapped, n=11 filled, 87-session evidence window)

## Finding
F-DNA1 (S367) asked: can explicit replication/mutation mechanisms close the Darwinian selection loop? It proposed P1 (selection, DONE) then P2-P7: classify_mutation, proofread, recombination. After 87 sessions of staleness, all three mechanism categories now exist — but built under different names:

- **Selection** (2/3): compact.py (archive low-quality), dispatch UCB1 (invest in high-value). genesis_selector.py was removed — selection shifted from offspring swarms to domain lanes.
- **Proofreading** (4/4): check.sh + validate_beliefs.py + correction_propagation.py + contract_check.py. Error correction pre- and post-replication.
- **Recombination** (3/3): knowledge_recombine.py (2,890 missing edges), frontier_crosslink.py (domain->global), historian_router.py (synthesis routing).

11/12 tool slots filled. The remaining gap: per-change mutation classification (point vs structural vs neutral). change_quality.py works at session-level, not knowledge-unit level.

Pattern: frontiers stall when framed as explicit construction projects (build X tool) but close through convergent evolution when the same function is needed by adjacent work. F-DNA1 expected a DNA toolkit; the swarm built the same functions as maintenance infrastructure. L-601 applies: the Darwinian loop closed structurally because selection/proofreading/recombination were USEFUL, not because they were PLANNED.

## Rule
Stale frontiers (>50 sessions) should be audited for convergent resolution: the mechanisms they requested may exist under different names. Map requested functions to current tools before declaring abandoned. Convergent closure is a valid resolution type.

Falsified if: removing any one of the convergent tools (compact.py, correction_propagation.py, knowledge_recombine.py) causes no measurable quality degradation, indicating they aren't actually performing the Darwinian function attributed to them.
