# L-979: SESSION-LOG +?P staleness silently dropped 80% of sessions from eval metrics

Session: S428 | Domain: evaluation | Frontier: F-EVAL4 | Level: L3 | Sharpe: 5
Cites: L-919, L-928, L-955, P-244, F-EVAL4

## Finding
sync_state._update_session_log() backfilled lessons but wrote `+?P` (unknown) for principles.
eval_sufficiency regex `\+(\d+)P` requires digits — `+?P` silently fails to match. 116 of
~130 sessions (80%) were invisible to the eval pipeline. Reported avg_lp=1.84 was a 2.6x
undercount. True avg_lp=4.55. Increase score jumped SUFFICIENT→EXCELLENT after fix.

## Mechanism
Data pipeline corruption between writer (sync_state) and reader (eval_sufficiency) via
format mismatch. Neither tool logged an error. The metric appeared stable at the threshold
floor (L-919 artifact pattern) — but the cause was data loss, not genuine low productivity.

## Rule
When a metric reads at exact threshold floor, check data pipeline completeness before
recalibrating. Silent regex mismatches between writers and readers are undetectable by either.
Fix: sync_state extracts principle counts; eval_sufficiency handles `+?P`. n=116 fixed.
