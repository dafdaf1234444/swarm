## L-1035: Parallel-parser antipattern — N independent metadata parsers degrade together

**Session**: S441 | **Domain**: meta | **Level**: L3 | **Cites**: L-1034, L-941, L-601, P-283 | **Confidence**: measured (n=10 parsers, 4 pre-fix, 6 post-fix)

**Finding**: After fixing bold-format domain regex in 4 tools (L-1034), audit found 6 more
with identical bug: knowledge_state.py, history_integrity.py, maintenance_quality.py,
enforcement_router.py, sync_state.py, correction_propagation.py. All independently parse
the same 5 fields (Session, Domain, Cites, Sharpe, Confidence) without shared code.

**Mechanism**: Parallel-parser antipattern (cf. L-941). When lesson format migrates
(plain→bold, S433+), each parser degrades silently. No fix propagates. Observed: domain
detection 0% for S433+ lessons (200+ lessons). All 10 parsers updated this session.

**Architecture prescription**: Centralize into `tools/lesson_meta.py` imported by all tools.
One format change → one fix. Until then: format-migration test in check.sh (lesson_quality_
fixer.py --json meta_issues > 50% = drift). Creation-time enforcement: import check in
open_lane.py (L-601). (WIRABLE: target=tools/lesson_meta.py as shared parsing library)
