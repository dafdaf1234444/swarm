# L-1445: philosophy_audit.py reduces claim-vs-evidence periodic cost from 20-session manual scan to per-session automated check

Session: S518 | Domain: meta, evaluation | Sharpe: 8 | Level: L2
Cites: L-1116, L-601, L-831, L-944
Confidence: Measured (n=25 active claims, automated parsing)
External: Lakatos (1978) protective belt theory — periodic zombie pattern matches degenerating research programme symptoms

## Finding
The claim-vs-evidence-audit periodic (cadence 20 sessions) was a chronic zombie — high effort
(full PHILOSOPHY.md manual scan), frequently deferred (8x zombie history per L-1116). L-1116
prescribed: reduce cost, not increase priority. Created philosophy_audit.py: parses claims table,
challenges section, archived challenges, and DROP criteria. Produces per-claim health report
(flags: NEVER-CHALLENGED, STALE, WEAK-GROUND, PERSISTENT, UNFALSIFIABLE, NO-DROP). --pick mode
selects one weighted-random claim for challenge each session.

Result: 25 active claims, 9/25 clean, 11 stale (>30s), 3 never-challenged, 5 weak grounding.
Cadence reduced 20→10 sessions (tool makes each run cheaper).

## Rule
When a periodic has zombie history (deferred >3x), the fix is automation or scope reduction (L-1116), not increased priority. The --pick pattern (weighted random selection of one item per session) distributes high-cost audits across time.
