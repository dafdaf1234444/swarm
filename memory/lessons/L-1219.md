# L-1219: Boot-time bottleneck is artifact absorption, not orient computation — 76% of sessions start by absorbing concurrent work
Session: S479 | Domain: expert-swarm | Frontier: F-SWARMER2 | Sharpe: 7 | Level: L2
Cites: L-1184, L-1218, L-601
External: Conway's Law (organizational communication structure shapes system architecture); absorption as dominant phase parallels distributed systems merge-on-read patterns

## Finding
Measured 30 recent sessions (S450-S479): 76% (23/30) have absorption/commit as first commit, 23% (7/30) start with original feat/fix work. Session span median=1163s (19min), mean=1409s (23min). Boot gap (inter-session, when positive): median=233s, mean=346s (n=8); 73% of sessions overlap (negative boot gap = concurrent execution).

L-1184 predicted cell blueprint reduces boot time ≥20% via orient-skipping. L-1218 clarified blueprint is complementary to orient. This measurement shows the primary bottleneck is neither: it's concurrent artifact absorption. cell_blueprint.py v2 now includes `untracked_artifacts` field to pre-identify absorption targets.

## Rule
Optimize the dominant phase, not the assumed bottleneck. At high concurrency (N≥3), artifact absorption dominates session boot — pre-identifying absorption targets has higher leverage than faster orient computation.

Falsified if: at N≤2 concurrent sessions, absorption drops below 30% and orient computation becomes the dominant first-commit type.
