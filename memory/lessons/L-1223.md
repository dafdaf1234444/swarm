# L-1223: Grounding audit false instrument — 57% "stale" claims had test dates the tool ignored
Session: S480 | Domain: meta, evaluation | Frontier: F-GND1 | Sharpe: 8 | Level: L3
Cites: L-1192, L-1204, L-1213, L-1216, F-GND1
External: none — measurement instrument fix, not external-contact claim

## Finding
grounding_audit.py (L-1192) showed 2.4% well-grounded, 57% stale claims. Investigation: PHILOSOPHY.md status column contains rich test histories ("CONFIRMED S394", "S458 REFINED", "S474 REFRAMED") but `_parse_phil_claims()` never extracted session numbers as `last_tested`. All 25 PHIL claims with S-references were scored as "never tested" (recency=0.2, floor). Fix: regex-extract highest S-number from status as last_tested. Also: axiom-type claims decay at 0.5× rate (definitional claims don't lose validity at the same rate as empirical measurements). Result: avg score 0.128→0.170 (+33%), stale 57%→21%, poorly-grounded 81%→69%. PHIL-17 jumped from 0.050 to 0.465 (tested S474). This is the third false instrument detected in grounding/evaluation tools (after L-1204 signal_density, L-1213 import chain).

## Rule
When a measurement tool shows catastrophically bad metrics, check the instrument before concluding the system is broken. Pattern: (1) tool shows alarming results, (2) investigation reveals tool doesn't parse available data, (3) fix the tool, (4) re-measure. Three consecutive false instruments (L-1204, L-1213, L-1223) in the grounding pipeline suggest: new monitoring tools need a calibration session before their output drives decisions.
