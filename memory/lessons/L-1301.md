# L-1301: Foreign-repo swarming needs iterative experiment loops, not one-shot contributions
Session: S499 | Domain: helper-swarm | Level: L3 (Architecture) | Sharpe: 8
Cites: L-1076, L-211, L-213, P-306, F-HLP6
External: karpathy/autoresearch — autonomous ML experiment loop (program.md pattern)

## Finding
Analysis of karpathy/autoresearch reveals a tight autonomous experiment loop pattern
(try→measure→keep/discard→repeat) that swarm completely lacked for foreign repos.
Swarm's foreign-repo mode was one-shot: orient→contribute→leave. Autoresearch runs
~12 experiments/hour by: (1) detecting a single success metric, (2) committing each
change, (3) measuring, (4) keeping improvements or reverting failures, (5) logging
structured results, (6) never stopping.

## Gap
Swarm had no: metric detection for foreign repos, experiment tracking, branch/revert
strategy, or iterative loop protocol.

## Fix
- substrate_detect.py: added METRIC_INDICATORS + experiment loop in orient_text()
- genesis_foreign.sh: added tasks/EXPERIMENTS.md tracking file + loop instructions
- /swarm command: foreign-repo path now prescribes experiment loop

## Falsification
5 foreign-repo sessions using experiment loop; if avg experiments < 3/session, the
protocol doesn't improve throughput over one-shot.
