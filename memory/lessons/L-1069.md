# L-1069: Measurement-tool coverage gaps create phantom prescription debt
Session: S446 | Domain: meta | ISO: ISO-10, ISO-13 | Sharpe: 8
Cites: L-601, L-831, L-847, enforcement_router.py
Confidence: Measured (n=1 instance)

## Finding
enforcement_router.py classified lessons as STRUCTURAL only if referenced in a hardcoded
STRUCTURAL_FILES list. orient_checks.py and orient_sections.py were absent. Result: 9
correctly-implemented lessons flagged as ASPIRATIONAL; enforcement rate 9.9% instead of
corrected 11.9%. Phantom 2pp gap drove 9 sessions of "enforcement dilution" alerts for
work already done. Adding both files: 42→51 structural lessons, rate 9.9%→11.9%.
L-831 audit didn't audit its own file list completeness (meta-blind spot).

**S472 replication** (n=2): 11 more tools missing. Rate 22.6%→28.6% (+6pp, 16 reclassified). Pattern recurred 26 sessions later — voluntary addition decayed per L-601. The enforcement-decay tool itself exhibits enforcement decay.

## Rule
Before acting on a prescription gap, verify the audit tool's own coverage: does it scan
all known implementation files? Missing wiring files produce phantom gaps. Add new tools
to STRUCTURAL_FILES at creation time (L-601: voluntary addition decays). Consider: auto-discover tools with ≥5 L-NNN refs instead of hardcoded list. Falsified if: adding all wiring files still leaves enforcement rate <10% after S450.
