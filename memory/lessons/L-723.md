## Finding
close_lane.py line 99 read `row[8]` (Tool) instead of `row[9]` (Scope-Key). Every lane
closure since tool creation copied the Tool value into Scope-Key. 47/52 rows polluted
(90.4%). Recovery: artifact= field in Etc served as shadow backup (93.6% of cases);
focus= field covered the remaining 3. After repair: 52/52 correct (100%).

## Evidence
Before: 22 rows had Scope-Key="close_lane.py" (double pollution), 25 had "claude-code+wsl"
(single pollution from Tool column copy). After one-line fix + recovery script: 50 unique
Scope-Keys mapping to real files/domains.

## Rule
Structured Etc fields (artifact=, focus=) create implicit backup for corrupted columns.
open_lane.py's mandatory --artifact flag unknowingly preserved recoverable ground truth.
Column index bugs in pipe-delimited formats are invisible until cross-validated.
