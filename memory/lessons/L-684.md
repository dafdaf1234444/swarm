# L-684: Swarm compaction is two-pathway append-only log, not Merkle tree
Confidence: Measured (n=661 lessons, 13 SUPERSEDED edges, 1070 citation edges)

**Observation**: Built SUPERSEDED DAG and citation DAG for full corpus. SUPERSEDED: 13
edges (5 L→L revision, 8 L→P promotion), 23 nodes, 10 components, max depth 1.
Citation: 1070 edges, 486 nodes, 5 components, max depth 41. Consumption rate 1.8%.
Production:compaction ratio 82:1 (citations vs supersessions).

**Result**: Merkle tree analogy is PARTIAL — chains exist (n=12) but all depth-1 (no
multi-hop compaction). Two pathways: horizontal revision L→L (38%) and vertical promotion
L→P (62%, e.g. P-153 absorbed 3 lessons). Citation transfer rate 0.6 supports Merkle
over flat model. But tree structure requires depth, which doesn't exist. The right crypto
formalization is append-only log with two-pathway garbage collection, not Merkle tree.
Proxy-K grows because collection rate (1.8%) << production rate.
