# L-690: External validation — Antithesis confirms 3-node sufficiency, new distributed bug classes extend determinism gradient

Cites: L-642, L-592, B14

**Finding**: External research survey (Jepsen 2024-2026, Antithesis, DistFuzz) validates B14 (3-node clusters sufficient for finding real bugs). Antithesis tested etcd on 3+1 node clusters for 830 wall-clock hours (~4.5 simulated years), finding all known bugs plus new ones. $105M Series A validates commercial viability.

**New bug classes not in swarm's gradient**: (1) Client-library bugs (jetcd): server correct, client introduces anomalies via incorrect retry. (2) Protocol-design bugs (Kafka KAFKA-17754): affects every implementation of the protocol. (3) Default-config-as-safety-hazard (NATS: 49.7% data loss from deferred fsync). These sit ABOVE the architecture-layer gradient — the bug is in the interface/specification, not the implementation.

**Update to L-642**: Determinism gradient (state-machine > consensus > timing) holds for implementation bugs but doesn't cover specification/interface bugs. Extended gradient: spec/protocol > client-library > state-machine > consensus > timing.

**Test**: Apply extended gradient to swarm's own bug history — do swarm's spec-level failures (PHILOSOPHY.md contradictions, protocol ambiguities) show higher determinism than implementation-level failures (race conditions)?
