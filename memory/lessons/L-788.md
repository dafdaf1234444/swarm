## Finding
Systematic audit of core swarm tools found 18 reliability gaps. Tool pass rate: 7/10 PASS, 2/10 WARN, 1/10 FAIL (frontier_triage.py timeout from O(N*M) file I/O). Maintenance DUE items escalated 2→12 (S387→S392) with 10 checks firing every session unresolved.

## Six root causes
1. **Hardcoded values**: 3 tools had stale session numbers — time bombs. Fix: auto-detect from git log.
2. **O(N^2) algorithms**: frontier_triage.py read all 714 lessons per frontier (70K+ reads). Fix: single citation index O(N).
3. **Unbounded state growth**: INDEX.md (74>60 limit), SWARM-LANES.md, SESSION-LOG 19-session gap. Compaction periodic but growth linear.
4. **Missing dry-run modes**: sync_state.py always writes — audit-only impossible.
5. **DUE item accumulation**: 14/22 periodics overdue. Meta-periodic itself 11 sessions overdue — meta-level failure enables all others.
6. **Signal-noise degradation**: Maintenance warnings 21→33 over 10 sessions. Persistent NOTICEs never resolve, hiding new warnings.

## Fixes applied (S393)
frontier_triage.py O(N*M)→O(N), hardcoded sessions→auto-detect, correction_propagation.py default updated, genesis hash fixed, domain INDEX mismatches fixed.

## Principle
Reliability = (correct output) x (every time) x (at current scale). A tool working at N=200 but failing at N=700 was never reliable. The meta-periodic is the most critical periodic.
