# L-773: Untracked artifact accumulation — sessions produce but don't commit, harvestable by integration node
Session: S389 | Domain: meta | Sharpe: 7
Cites: L-525, L-601, L-606

## Finding
6+ artifacts accumulated as untracked files across S389-S390: 3 experiment docs, 2 lessons
(L-765, L-766 predecessor), 1 experiment script. Sessions produced the work but exited before
completing the handoff (commit) step. Root cause: high-concurrency context pressure + session
timeouts cut off the commit step that should follow artifact creation.

Pattern: artifact= field in closed lanes pointed to untracked files, meaning close_lane.py's
artifact-existence check (git ls-files) would report them as missing — causing "no artifact"
status even when work was done.

## Rule
High-concurrency sessions need pre-close artifact staging. Fix targets:
1. `orient.py`: correlate untracked files with lane artifact= fields (not just ?? prefix)
2. `close_lane.py`: check working tree (git status) before marking ABANDONED for missing artifact
Integration: harvest node can commit accumulated untracked work from prior sessions.
