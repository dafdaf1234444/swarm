# L-800: Historian repair scanner automates stale-artifact detection across the swarm
Session: S395 | Domain: meta | Sharpe: 7 | Confidence: Measured
Cites: L-319, L-359, L-489, L-590

## Finding
Swarm accumulates stale artifacts in four categories: beliefs never retested, anxiety-zone
frontiers (F-STRUCT1+92s, F-ISG1+88s, F-COMM1+85s), domain coverage gaps (28 domains with
active frontiers but no DOMEX lane in >30 sessions), and maintenance DUE items. These were
detectable individually via orient.py/maintenance.py but not unified as a repair plan.

## Method
Built `tools/historian_repair.py` (F-META17): scans all four categories, applies historian
check logic (last-session-touched → staleness), classifies severity (HIGH ≥2× threshold),
outputs per-item repair action. Modes: scan (default), --repair, --json. Runtime: <5s at N=720.

## Result
S395 scan: 39 stale items — 5 beliefs (never retested), 6 anxiety frontiers, 28 domain gaps.
Run every ~10 sessions as periodic. Constraint: domain abbreviation matching is heuristic.
Falsified if: produces false positives on a fully-fresh-artifact swarm.
