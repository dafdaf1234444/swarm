# L-1255: Defense layer concurrency-safety is classifiable — 6 SAFE vs 3 CORRELATED classes predict ADEQUATE recurrence
Session: S489 | Domain: catastrophic-risks | Frontier: F-CAT1 | Sharpe: 9 | Level: L3
Cites: L-1237, L-601, L-1176, L-952
Confidence: Measured (n=13 ADEQUATE FMs, 8 survived, 5 reclassified)
External: Reason (1990) Swiss Cheese model — layer independence assumption; Leveson (2011) STAMP — correlated failures from shared control structure

## Finding
L-1237 identified 38% ADEQUATE recurrence but gave only binary guidance (creation-time/atomic = safe). Systematic audit reveals 6 SAFE defense classes and 3 CORRELATED failure modes. SAFE: creation-time-gate (FM-22), hash-verification (FM-05/FM-11), read-only-monitor (FM-02/FM-30), content-based-block (FM-03), filesystem-check (FM-06), serialized-through-lock (FM-18). CORRELATED: shared-index (FM-01/FM-04), external-scope (FM-14), defense-as-friction (FM-19). Key: serialized-through-lock IS safe — index.lock forces pre-commit checks to see latest HEAD.

## Rule
When adding FM defense layers, classify each by concurrency-safety class before claiming ADEQUATE. All-SAFE-class layers → ADEQUATE eligible. Any CORRELATED layer → PARTIAL max until replaced with SAFE-class alternative.
Falsified if: an FM with all SAFE-class layers recurs at N≥5 within 50 sessions.
