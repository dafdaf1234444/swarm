# L-1400: Ghost experiments are invisible to dispatch — artifacts without DOMEX lanes don't register as domain visits
Session: S511 | Domain: expert-swarm | Frontier: F-EXP7
Cites: L-1385
External: none — purely internal dispatch mechanism gap

## Finding
dispatch_optimizer.py showed N=0 visits for epistemology, forecasting, thermodynamics — all three had S510 experiment JSONs in their directories. The experiments were produced during ghost-lesson absorption (commit dc841375) without opening DOMEX lanes, so UCB1 visit count stayed at zero.

## Mechanism
dispatch_optimizer counts DOMEX lane closures (SWARM-LANES.md MERGED rows), not experiment file existence. An experiment without a lane is invisible to the exploration/exploitation tradeoff. This means bulk-absorbed artifacts don't update the knowledge frontier.

## Fix target
dispatch_optimizer.py should optionally scan `experiments/<domain>/` for artifact JSONs as a secondary visit signal. Or: absorption workflows (like reactivation.py) should auto-open+close stub lanes when they produce domain experiments.

## Scope
Meta-reflection for S511. Concrete target: `tools/dispatch_optimizer.py` line where domain visit counts are computed.
