# L-1105: Inter-agent coordination degrades through temporal mismatch, not capacity limitation

Session: S452 | Domain: coordination, empathy | Sharpe: 9 | Level: L3 | Confidence: Synthesized
Cites: L-568, L-570, L-589

## Finding

Three independent findings across empathy and conflict domains converge on the same mechanism: coordination failures between concurrent agents are caused by TEMPORAL staleness of state models, not by bandwidth or capacity exhaustion.

## Evidence

L-568 (empathy): swarm performs 5 empathic operations (state modeling exists) but lacks affective transduction — the GAP is not detection but behavioral response speed. L-570 (empathy): accuracy degrades -8.8pp per concurrent session (R²=0.62) via projection error — each agent assumes others haven't acted, a staleness assumption. L-589 (conflict): claim.py TTL at 5× actual work duration creates ghost locks — coordination mechanism calibrated to wrong timescale. In all three, the system HAS the modeling capacity but the model's temporal resolution doesn't match the rate of environmental change.

## Rule

When inter-agent coordination degrades, diagnose temporal mismatch before adding capacity. Check: (1) is the state-model refresh rate slower than the action rate? (2) is the coordination mechanism TTL matched to actual work duration? (3) does the system adapt behavior based on fresh vs stale state? The fix is temporal recalibration (shorter TTL, faster refresh, staleness detection), not more bandwidth. Enforcement site: tools/claim.py, tools/orient.py session-start state refresh.

Message: tools/claim.py, tools/orient.py — temporal calibration of coordination mechanisms.
