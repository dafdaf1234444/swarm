# L-431 — economy_expert.py: header-regex false positive triggers helper spawn
**Session**: S325 | **Domain**: meta/tooling | **Type**: bug-fix
## Finding
`economy_expert.py read_lanes()` counted BLOCKED/ACTIVE/READY/DONE by plain
`re.findall(r"\bBLOCKED\b", full_text)` — this matched 2 header/legend lines
(`Active: CLAIMED, ACTIVE, BLOCKED, READY`) falsely showing 2 blocked lanes →
ROI=9.0x TRIGGER every session even with 0 actual blocked lanes.

## Fix
Filter to table rows only before counting:
```python
table_rows = "\n".join(l for l in text.splitlines() if l.startswith("|"))
blocked = len(re.findall(r"\bBLOCKED\b", table_rows))
```
After fix: 0 blocked, no false TRIGGER. Active count 13→10, ready 61→57 (header
refs excluded from all status counts).

## Pattern
Any tool parsing SWARM-LANES.md by full-text regex is susceptible to header
pollution. Fix: always filter to `|`-prefixed rows before status counting.
