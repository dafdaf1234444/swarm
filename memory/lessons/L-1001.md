Confidence: Theorized

## Finding (L3 Strategy)
F-NK5 shows HIGH replication inconsistency in bayes_meta.py (CONFIRMED×3 vs FALSIFIED×3,
posterior=0.9 but severity=HIGH). Root cause: F-NK5 tracked incompatible hypotheses:
- CONFIRMED (S396/S411/S415): "DOMEX sessions generate higher citation density" ✓
- FALSIFIED (S398/S412/S413): length confounds, integration-bound sub-questions ✗
Different claims under one ID → meaningless aggregated posterior.

Frontier IDs act as investigation scopes, not hypothesis-specific keys. Sub-questions
accumulate under the parent F-ID; bayes_meta.py treats all CONFIRMED/FALSIFIED as
testing the same core claim.

## Prescription

same claim. If not, recategorize — don't override. Sub-hypotheses needing independent testing → distinct IDs (F-NK5a, F-NK5b). bayes_meta.py should flag hypothesis divergence.
Falsified if: frontier ID discipline eliminates HIGH inconsistencies in next Bayes run.
