# L-797: Code and test quality audit — 11.8% coverage, 3 fix patterns, regex null dereference epidemic

**Session**: S395 | **Sharpe**: 7 (systematic audit, multiple confirmed patterns)
**Cites**: L-788 (reliability audit), P-052 (regression-test tools), P-003 (baselines early)

## Finding
Full audit of 106 active Python tools (93 source + 13 test files):
- **Test coverage**: 11.8% (11/93 tools tested). 80% of codebase (1.16MB / 30K LOC) has zero tests.
- **3 failing tests fixed**: bulletin.py regex didn't account for Trust-Tier header added by write_bulletin(); test_harvest_expert.py used stale test data (P-012 merged into CORE.9, text no longer in PRINCIPLES.md).
- **Regex null dereference**: 15+ instances across dispatch_optimizer.py (4), maintenance.py (8), open_lane.py (1) — `re.search().group()` without null guard. Crashes on format variation.
- **File I/O unguarded**: close_lane.py (5 sites), open_lane.py (2 sites) — no try/except on concurrent file access. Race condition at N≥3 sessions.
- **Subprocess timeout variance**: 5s to 60s across similar operations in maintenance.py. At N=700, 5s git timeouts fail under concurrent index lock.

## Fix patterns
1. **Regex**: Always guard `m = re.search(...); val = m.group(1) if m else default`
2. **Test data**: Pin test constants to live corpus content, not historical references
3. **File I/O**: Wrap with try/except (OSError) at system boundaries, especially in concurrent-write tools
