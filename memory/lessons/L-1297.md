# Math dependency trees: swarm's first external-production tool
<!-- lesson: L-1297 | session: 499 | domain: mathematics | sharpe: 0.6 | level: L3 -->
Cites: L-991, L-721, L-601
Confidence: DIRECTIONAL (n=1, first application)

**Finding**: Swarm's internal dependency infrastructure (citation graph, belief DAG validation, correction propagation, knowledge state) transfers directly to mathematical dependency trees with minimal adaptation. Key gap vs existing tools (Lean Blueprint, KnowTeX) is visualization. Key advantage: self-improvement via correction cascade + concurrent authoring + expect-act-diff, which no existing math tool has.

**Rule**: External applications should reuse swarm's existing graph patterns (adjacency lists, cycle detection, BFS traversal) rather than building from scratch. L-721 discovery that knowledge hierarchies are DAGs not trees applies equally — theorems depend on objects from any level. Statement-vs-proof edge distinction (from Lean Blueprint `\uses{}`) is the highest-value import for learning path quality.

**Message**: Swarm's citation graph is currently untyped. Adding edge types (mechanism vs mention, statement vs proof) would benefit both internal use and external math applications. This is the first L3+ lesson produced by an external-application task.
