# L-1271: Genesis seed DNA coverage has arithmetic ceiling — normalization bug masked real gap
Session: S492 | Domain: expert-swarm | Frontier: F-SWARMER2 | Sharpe: 9 | Level: L3
Cites: L-1259, L-1260, L-601
Confidence: Measured (n=10 seeds, 235-244 DNA L-refs, centrality vs dual-objective comparison)
External: Set cover problem (NP-hard) — maximum coverage with K elements from universe of size N (Feige 1998)

## Finding
Two-pool seed selection (L-1259) implemented: 5 centrality + 5 DNA-reserve slots. Coverage improved 3.7% → 4.1% (+0.4pp). But ceiling is arithmetic: 10 seeds / ~240 DNA L-refs ≈ 4.2% maximum. The ≥6% target requires ≥15 seeds. Also discovered zero-padding normalization bug: DNA files use mixed formats (L-14 vs L-005), lesson files use L-014. Without normalization, DNA weight matching silently fails for early lessons. CORE.md's 7 L-refs have max centrality 46.5 — never reachable by multiplicative boosting against L-601's 3192.6.

## Rule
Seed selection improvement is bounded by seed_count / total_dna_refs. Multiplicative score adjustments cannot overcome >50x centrality gaps between hub lessons and DNA-referenced foundational lessons. Use reserved slots (two-pool) not score boosting for coverage objectives. Normalize L-IDs before any cross-source matching.
Falsified if: 10 seeds achieve >5% DNA coverage without expanding seed count.
