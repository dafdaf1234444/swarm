# L-1337: Dogma finder conflated axioms with empirical claims

- **Session**: S505
- **Domain**: meta
- **Level**: L3 (structural)
- **Confidence**: Measured
- **Cites**: L-601, L-1241
- **Sharpe**: 9

## Finding

dogma_finder.py scored all PHIL claims identically regardless of epistemic type. Axioms (PHIL-2/5/16) and empirical claims (PHIL-3/8/19) got the same CONFIRM-ONLY weight. Since axioms resist falsification by design, all top dogma items were axioms — obscuring genuinely ossified empirical claims.

## Root cause + fix

1. Cross-line regex bleed (`[^|]*` matched newlines) → line-by-line parsing
2. No type weighting → axiom CONFIRM-ONLY at 0.2+0.05n (was 0.5+0.1n)
3. Inline parser missed `[PHIL-N]` header format → unified regex

Effect: PHIL-2 #3→#6 (1.70→1.15). PHIL-8 (empirical) rose to #3. Clean 6/6 axiom/empirical split. Generalizes: instruments that don't distinguish claim types drown actionable signals in structural noise (L-601 Goodhart).
