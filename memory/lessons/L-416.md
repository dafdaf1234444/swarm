# L-416: IFS=$'\n' read with here-string reads only first line
**Domain**: swarm-ops | **check_mode**: verification

## Bug
`IFS=$'\n' read -r v1 v2 v3 <<<"$multi_line"` reads only the FIRST line.
All subsequent variables are empty. Used in `tools/autoswarm.sh` to parse
Python output â€” `ANXIETY_FRONTIER` always empty, dry-run exited with
"missing frontier id". The anxiety-gate appeared wired (S307) but silently
failed every invocation.

## Fix
`mapfile -t arr <<<"$str"` + index by position:
```bash
mapfile -t _fields <<<"$ANXIETY_PARSED"
ANXIETY_STATUS="${_fields[0]:-}"
ANXIETY_FRONTIER="${_fields[1]:-}"
```

## Validation
`bash tools/autoswarm.sh --dry-run`: `prompt_source=anxiety_gate, frontier=F111`.
Stop hook: `touch workspace/autoswarm-trigger`. (bash, autoswarm, F-COMM1)
