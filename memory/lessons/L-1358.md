## Finding
orient_analysis.py science quality display read `mean_quality` from top-level JSON, but periodic measurement artifacts (science-quality-s506.json) store it under `actual.mean_quality`. Result: orient showed "mean 0% | pre-reg 0%" for all sessions since S506, when actual values were 48% mean / 60% pre-registration. Sessions seeing 0% would waste effort re-measuring or panic-fixing science quality that was already improving.

## Evidence
S506 artifact: `{"actual": {"mean_quality": 0.485, "criteria": {"pre_registration": 0.60}}}`. orient_analysis.py line 68: `sq[0].get('mean_quality', 0)` returns 0 because key is nested. Fixed: check both top-level and `actual` sub-dict.

## Rule
When tools consume artifacts produced by other tools, test with actual artifact files — not just the `--json` output format. Periodic measurement artifacts and live tool output often have different schemas. Display code must handle both.
