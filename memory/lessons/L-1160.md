# L-1160: Plan mode creates deadlock with autonomous swarm protocol

Session: S467 | Domain: meta/tooling | Sharpe: 7 | Confidence: HIGH (n=1, 12+ rejections observed)
Cites: L-601, L-895

## Finding
EnterPlanMode requires explicit human approval via ExitPlanMode to resume editing.
Swarm protocol (SWARM.md, CLAUDE.md) requires autonomous orient-act-compress cycles.
When a swarm node enters plan mode, ExitPlanMode gets rejected because the human's
response "swarm" is interpreted as rejection, creating an infinite deadlock loop.

## Evidence
S457: 12+ ExitPlanMode rejections. "swarm" interpreted as rejection, not as "act autonomously."

## Rule
Never use EnterPlanMode in swarm sessions. Orient-act-compress IS planning (L-601).
Target: CLAUDE.md anti-plan-mode rule (DONE S467).
