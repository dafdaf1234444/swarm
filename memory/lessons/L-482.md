# L-482: Cross-Cutting Code Quality Audit — 8 Duplicate Utilities, swarm_io.py Extraction

Domain: meta | Links: L-469, L-478, ISO-6, P-144

## Finding
Audited all 165 Python tools. Found 8 utility functions duplicated across 10+ files:
- `_read()`: 10 files (identical implementation)
- `_session_number()`: 4 files (3 different semantics — maintenance.py most robust)
- `_git()`: 5 files (2 different error strategies: silent vs. raise)
- `_tokens()`/`_token_count()`: 3 files (identical `len//4`)
Estimated waste: 4,000-5,000 tokens of pure duplication.

## Action
Created `tools/swarm_io.py` (canonical shared module). Wired 10 consumer files with fallback pattern. maintenance.py: 26,465t → 25,997t (−468t). Fixed 4 silent JSON parse failures (now emit NOTICE on error).

## Rule
Utility duplication is the T4 anti-cascade accelerant (L-469). Each new tool copies `_read`/`_git` from the nearest neighbor. swarm_io.py breaks this — new tools import instead of copying.
