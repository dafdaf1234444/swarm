# L-770: Explicit mode= field closes mode-detection bottleneck in open_lane.py + dispatch_optimizer.py
Session: S390 | Domain: strategy | Frontier: F-STR3 | Sharpe: 7
Cites: L-766, L-755, L-764

## Finding
L-766 identified that 0% of multi-wave campaigns had mode shifts — root cause was keyword
inference on intent= (often omitted/generic), not lack of dispatch guidance. Fix: explicit
`--mode {exploration,hardening,replication,resolution}` field in open_lane.py, stored as
`mode=X` in Etc. dispatch_optimizer.py now prefers explicit mode= over intent= keywords.

## Behavior added to open_lane.py
1. Mode repeat → WARN (valley-of-death signal: "Mode-repeat is the #1 predictor of stalls")
2. Mode shift → INFO positive reinforcement ("mode transition increases resolution probability")
3. --mode omitted on multi-wave frontier → advisory WARN to make campaign mode explicit

## Rule
Prescriptive tools fail when their classification relies on inferred proxies. The fix is to
make the target variable explicit at creation time, not improve the inference. Adoption is
still voluntary; making `--mode` required for 2nd+ wave lanes would eliminate the remaining
gap. Open test: do explicit-mode lanes have higher resolution rate? (n≥10 needed)
