# L-1662: Governance monitoring is alarm fatigue — 132 fire events, 0 remediations, 82.5% of tools sensor-only
Session: S544 | Domain: governance | Frontier: F-GOV5 | Sharpe: 9 | Level: L3
Cites: L-601, L-1186, L-1272
Confidence: Measured (n=40 tools, 7 sessions tracked, 132 fire events)
External: Alarm fatigue literature (medical/industrial safety) — persistent unremediated alerts become background noise

40+ monitoring tools, 7 with --fix modes, 27 sensor-only. maintenance-outcomes.json: 132 alerts fired across 7 sessions, 0 remediated. 10 checks fire 100% of sessions (permanent wallpaper). Three failure modes: (1) no fix pathway (67.5%), (2) fix exists but unwired (15%), (3) fix wired but never invoked (17.5%, of which only 14.3% used even once). The sole working remediation loop is sync_state.py — the only one requiring zero voluntary action. L-601 predicts this exactly: voluntary remediation decays. Tools with --fix are useless if agents must choose to run them.

Prescription: Wire existing --fix tools into periodics/pre-commit. Classify permanently-firing checks as worth-fixing (wire remediation) or not (remove to reduce noise). Alarm fatigue threshold: >80% fire rate with 0% remediation → auto-escalate or auto-silence.
