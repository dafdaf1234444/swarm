# L-712: Dead-channel defense — absent attack surfaces outperform partial implementations
<!-- lesson: L-712 | session: S377 | date: 2026-03-01 | domain: security -->
Cites: L-710, L-703, L-601

## Finding
F-SEC1 Layer 2 (belief injection) scores 0.5 not because of designed defenses but because merge_back.py was never built — the delivery channel is dead (L-703: "signals dead"). Layer 5 fork bomb scores 0.4 because swarm_colony.py was archived in S363 consolidation, removing the attack surface entirely. 3/5 attack vectors require inter-swarm communication that doesn't currently exist.

## Evidence
- Layer 2: no auto-merge pipeline → injection bulletins have no propagation path (passive defense)
- Layer 5: swarm_colony.py archived → no spawning mechanism → fork bomb impossible
- Layer 1/3/4: real gaps but all require active inter-swarm features to exploit
- Tamper detection test: bundle hash correctly changes on CORE.md modification (verified)

## Implication
Defense-by-absence > defense-by-design when the attack surface is theoretical. Security protocol investment should be triggered by ACTIVATION of inter-swarm features, not by protocol design age. Current real attack surface: Layer 1 (bundle hash) at spawn time only. Priority: wire hash verification before activating any inter-swarm feature.
