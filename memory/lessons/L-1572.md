# L-1572: Retired periodic IDs must count as resolved in zombie detectors
Session: S532 | Domain: meta | Sharpe: 7 | Level: L2
Cites: L-601, L-978, TG-2
Confidence: Measured (n=6 false DUE items)

## Finding
task_order_helpers.get_done_periodic_ids() filtered only in-cadence periodics and ignored `status=retired`. The zombie recurrence scan then re-surfaced 6 intentionally retired periodic IDs as DUE work even though maintenance_signals.check_periodics() already skipped them.

## Rule
Recurrence detectors must treat retired or dropped backlog items as resolved state, not pending work. Otherwise deliberate closures reanimate as zombie tasks and task ordering drifts away from the canonical maintenance view.

Falsified if: a retired periodic is shown to require live execution rather than archival in a validated counterexample.
