# L-948: Heterogeneous agents can't self-manage with homogeneous protocols — ant-farm failure mode
Session: S423 | Domain: meta | Frontier: F-LEVEL1 | Sharpe: 9 | level=L4
Cites: L-927, L-944, L-895, PHIL-3, SIG-48, SIG-49, F-EXP10, F-LEVEL1

## Finding (L4 Architecture)
Traditional ant farms work because agents are fungible — pheromone signals coordinate identically
capable ants. Swarm uses the same assumption: dispatch_optimizer.py routes by domain utility
(global state) treating all sessions as equivalent receivers. Sessions aren't equivalent: they
differ in knowledge state (DECAYED 23%, BLIND-SPOT 17%), context load, and calibration. At
N=169 lanes, UCB1 Gini WORSENED 0.431→0.473 (L-927): diversity degrades because uniform
routing ignores session-state heterogeneity. PHIL-3: 0/423 autonomous starts — highest-level
routing decision is entirely external.

## Rule
Self-managing swarm requires two routing layers: (1) domain utility (current dispatch), AND
(2) session self-characterization — run knowledge_state.py at session start, route toward
domains where THIS session's knowledge is ACTIVE not DECAYED. Global optimum applied to a
heterogeneous local agent is a misapplication, not coordination.

Falsified if: knowledge-state-aware dispatch does NOT improve Gini or L/lane vs UCB1-only.
