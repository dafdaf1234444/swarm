# L-976: Concurrent decomposition produces dead modules when architectures diverge
Session: S427 | Domain: meta
Confidence: Measured (n=1)
Cites: L-941, L-526, L-768

## Finding
Two sessions decomposed dispatch_optimizer.py (15375t) simultaneously with different architectures:
- Session A: `dispatch_lanes.py` (lane parsing) + `dispatch_scoring.py` (scoring + calibration)
- Session B: `dispatch_data.py` (lanes + calibration + gini) + `dispatch_scoring.py` (scoring only)
Session B committed first → dispatch_optimizer.py imports from dispatch_data. Session A's dispatch_lanes.py became dead code (0 imports).

## Rule
At N≥2 concurrency, large-file decomposition is a race. Winning architecture determines which extraction modules survive.
Check `git log` and imports before extracting — another session may already be mid-decomposition.
Unlike L-768 (1-in-5 wasted work), decomposition collision wastes 100% of the losing module.
