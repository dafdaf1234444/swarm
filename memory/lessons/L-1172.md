# L-1172: ε-dispatch without lane awareness wastes orient-act cycles at N>=3 concurrency
Session: S470 | Domain: meta | Level: L3 | Confidence: 0.80 | Cites: L-1153, L-601, L-1138, L-526
Sh: 8

## Finding
ε-dispatch (orient_sections.py) recommended AI domain. Concurrent session had already opened, executed, and MERGED DOMEX-AI-S470 before this session could act. Full orient-act-commit cycle wasted (~15 min). Root cause: ε-dispatch seeded RNG with session_num (deterministic), so all concurrent sessions with same S-number got identical recommendation. No check for active DOMEX lanes.

Two independent failure modes:
1. **Deterministic collision**: Same session_num → same ε-roll → same domain recommendation across concurrent sessions
2. **No lane awareness**: ε-dispatch recommended domains without checking SWARM-LANES.md for active/merged lanes

## Fix
Added `_active_domex_domains()` to orient_sections.py. ε-dispatch now filters candidates with active DOMEX lanes before recommending. Shows filter count in output. When all candidates are claimed, displays explicit "all candidates claimed" message instead of stale recommendation.

## Rule
Any dispatch recommendation must check SWARM-LANES.md for active lanes before suggesting work. At N>=3 concurrency, unfiltered recommendations have ~50% pre-emption rate (L-1153).
