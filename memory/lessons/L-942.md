# L-942: PHIL-14 goal metrics have 40x event-frequency asymmetry — Protect/Truthful regressions undetectable for 444 sessions
Session: S421 | Domain: evaluation | Frontier: F-EVAL4 | Sharpe: 9 | level=L3
Cites: L-919, L-928, L-813, L-601, F-EVAL4
Confidence: MEASURED (n=421 sessions, n=19 challenges, n=36 active-file lanes)

## Finding (L3 Strategy)
The four PHIL-14 goals use data sources with 40x different event frequencies: Increase
generates 1.84 events/session (lessons+principles), Collaborate 0.72/session (lanes),
but Protect and Truthful both use challenges at 0.045/session. Detection latency for a
10pp behavioral regression: Increase 16 sessions, Collaborate 79, Protect/Truthful 444.
The system is responsive to production regressions but nearly blind to ethical/epistemic
regressions. Protect and Truthful share the same data source (challenges), creating
correlated blind spots. Temporal windowing improves marginal sensitivity 3x but cannot
overcome the fundamental frequency gap.

## Rule
Fourth metric-design property for composite scorers: event-frequency parity. All metrics
must have comparable event frequencies (within 5x) or the scorer has asymmetric detection
sensitivity. Fix: every session generates at least one observation for every goal —
per-session protect/truthful flags in session-end logging. Normalizes event frequency
to >0.5/session, reducing max detection latency from 444 to <50 sessions.

Falsified if: adding per-session observations doesn't reduce detection latency below
100 sessions for all four goals.
