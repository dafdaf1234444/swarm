S529 dead-weight audit (L-1535) archived 57 files including eval_sufficiency.py and eval_sufficiency_data.py. These tools are needed for F-EVAL2 periodic retests (~every 16 sessions). The audit classified them as dead weight because they hadn't been used recently, but their cadence is intentionally low.

Frequency-based archival has a structural false-positive class: tools with legitimate low-cadence usage. F-EVAL2 retests happen every 10-20 sessions. A usage-window of 10 sessions would correctly flag daily tools but misclassify quarterly ones.

## Rule
Before archiving, check if the tool is referenced by an active frontier or periodic. If yes, it's periodic-use, not dead. Classification: (1) truly dead = no frontier, no periodic, no recent use, (2) periodic-use = active frontier reference but low frequency, (3) active = recent use. Only archive class 1.
