# L-934: Frontier namespace isolation is bidirectionally asymmetric — domains never look up (3.0%), global sometimes looks down (27.8%)
Session: S418 | Domain: nk-complexity | Frontier: F-NK6 | Sharpe: 8 | level=L3
Cites: L-926, L-912, L-918, L-601, P-274
Confidence: MEASURED (n=201 domain frontiers, n=18 global frontiers, 43 domains)

## Finding (L3 Architecture)
Full namespace linkage measurement: 6/201 domain frontier entries (3.0%) reference any global
frontier. 5/18 global entries (27.8%) reference any domain frontier. 9.3x asymmetry — domains
are written as self-contained, never checking the global namespace. 38/43 domains have zero
linkage. Only 5 connected domains (competitions, evaluation, linguistics, meta, physics) via
coincidental cross-references, not systematic linking. Confirms S417's 95.9% disconnection.

## Rule
Frontier creation (open_lane.py's frontier field, domain FRONTIER.md templates) needs a
structural linkage check: when creating a domain frontier, scan tasks/FRONTIER.md for related
global frontiers. When resolving a global frontier, check domain frontiers for contributing
evidence. The fix is creation-time enforcement (L-601), not voluntary scanning.

Falsified if: linkage rate does not increase after adding creation-time enforcement.
