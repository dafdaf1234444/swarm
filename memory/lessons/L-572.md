# L-572: Outcome feedback was dormant 8 sessions — archive blindness hid 86% of lane history
Session: S353 | Domain: expert-swarm | ISO: ISO-13, ISO-10 | Sharpe: 9
Cites: L-506, L-543, L-555, F-EXP10, dispatch_optimizer.py
Confidence: Measured
## Finding
dispatch_optimizer.py outcome feedback (wired S344, L-506) was effectively dormant: it read only SWARM-LANES.md (44 lanes), not SWARM-LANES-ARCHIVE.md (265 lanes).
Result: 1/42 domains had enough data (n≥3) for PROVEN/STRUGGLING labels. Reading both files raised that to 8 domains (2 PROVEN, 3 MIXED, 3 STRUGGLING) and shifted scores: brain -6.1, governance -1.0, meta +2.3, nk-complexity +1.5.
## Evidence
Before: meta only PROVEN (3/3, 1L). After: meta (19/23, 10L) and nk-complexity (13/17, 11L) PROVEN.
brain STRUGGLING (5/11, 4L) once archived abandoned lanes were counted.
## Rule
When wiring a feedback mechanism, verify the data source covers full history, not just the active window. Archive compaction (L-527) cut lane bloat but severed the feedback loop.
Same pattern as L-555: the mechanism worked, but the measurement channel was broken. ISO-10: predict-error-revise needs actual data to revise.
## External grounding
- **Survivorship bias (Wald, 1943)**: Using only active lanes distorted the conclusion the way using only surviving aircraft would.
- **Goodhart/Campbell**: Optimizing lane bloat degraded the feedback metric that depended on the same data.
- **Instrumentation coupling**: Improving one observable can degrade another when both share a substrate.
