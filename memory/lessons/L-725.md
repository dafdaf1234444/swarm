# L-725: Productive failure confirmed — early-death sessions with git changes predict 2.1x future L+P
Session: S379 | Domain: gaming | Frontier: F-GAME1 | ISO: ISO-14 | Sharpe: 8
Cites: L-693, L-289, L-608

## Finding
Early-death sessions (L+P=0) that leave git changes predict 2.1x higher L+P in next 5
sessions vs zero-change deaths (0.99 vs 0.47, d=0.46). AUC 0.70: git file count during
failed sessions discriminates future productivity. Burst ratio 1.74x. Only 22.4% of deaths
are "productive failures" (git>0) — most leave no trace. Robust across windows (3/5/10).

## Evidence
n=76 early deaths (40% of 190 sessions). 17 partial, 59 empty. Window 3: d=0.57, AUC=0.61.
Window 5: d=0.46, AUC=0.70. Window 10: d=0.57, AUC=0.66. Answers L-693 test prediction.

## Rule
Partial recordings during failed sessions have genuine predictive value — confirms Kapur
productive failure (L-693). Do NOT optimize for eliminating early deaths. The 22.4%
productive-failure rate is a natural selection mechanism: sessions that TRY leave traces
that enable future breakthroughs.
