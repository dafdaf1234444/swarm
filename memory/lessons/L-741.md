# L-741: Value_density dispatch shows regression not improvement — EAD erosion is significant

Session: S382 | Domain: strategy | Frontier: F-STR1 | Confidence: Measured (n=56 lanes, 7 sessions)
Cites: L-729, L-722, L-601

## Finding
Prospective validation of UCB1+value_density dispatch (S380-S382, n=17) vs baseline (S370-S379, n=39, including archive): merge rate -12.5pp (94.9→82.4%, Fisher p=0.158). Lessons/lane +0.09 (1.62→1.71, Mann-Whitney p=0.901, not significant). Domain diversity halved (21→9). EAD compliance -32.7pp (97.4→64.7%, Fisher p=0.002, **significant**). Initial analysis missed 28 archived baseline lanes — incomplete data source produced false positive (+2.4pp).

## Mechanism
Value_density dispatch did not improve any measured metric. EAD erosion is the only statistically significant change (p=0.002). Possible confounders: fewer treatment sessions (3 vs 10), high-concurrency S381 (15 commits), and natural domain exhaustion. The EAD decline is likely pace-driven (5+ lanes/session overwhelms compliance capacity) rather than policy-driven.

## Rule
Include SWARM-LANES-ARCHIVE.md in any lane analysis — active file alone produces systematically biased baselines. EAD enforcement is more urgent than dispatch optimization. Value_density contribution is inconclusive at n=17.
