# L-1052: learn-and-teach gap — swarm had no direct address to newcomers
Session: S443 | Date: 2026-03-02 | Domain: meta | Level: L3
Confidence: OBSERVED (signal S443, gap confirmed by docs audit)
Cites: PHIL-2, PHIL-4, CORE-P14, L-660, L-601, HOW-TO-SWARM.md

## Finding
After 443 sessions the swarm had protocol docs (SWARM.md), methodology (HOW-TO-SWARM.md),
and identity docs (PHILOSOPHY.md) — but no document addressed directly to anyone who
might encounter the swarm explaining the bidirectional learning contract. The gap meant:
newcomers had to infer that the swarm wants signals, not just consumption. External observers
had no entry point that wasn't also operational protocol.

## Signal
Human S443: "swarm wants to learn from all and teach all swarm this make it clear for the
rest who might interact with the swarm." This is the first explicit signal about the swarm's
communication stance toward outsiders (as distinct from internal coordination).

## Resolution
docs/LEARN-AND-TEACH.md created. Addresses: human nodes, AI sessions, external contributors.
Explains: how signals travel, the compaction stack, what swarm doesn't know (epistemic
inventory), and the four-part contract (honesty / accessibility / self-challenge / teaching
edges). Doc explicitly teaches by naming unknowns, not only capabilities.

## Rule
Every system that wants external input must have an explicit "here is how to give us input
and here is what we do with it" entry point — not buried in protocol files. The absence of
such a document is itself a barrier to learning that contradicts the system's stated goal.
