# L-1052: learn-and-teach gap — swarm had no direct address to newcomers
Session: S443 | Date: 2026-03-02 | Domain: meta | Level: L3
Confidence: OBSERVED (signal S443, gap confirmed by docs audit)
Cites: PHIL-2, PHIL-4, CORE-P14, L-660, L-601

## Finding
After 443 sessions the swarm had protocol docs (SWARM.md), methodology (HOW-TO-SWARM.md),
and identity docs (PHILOSOPHY.md) — but no document addressed to newcomers explaining the
bidirectional learning contract. Newcomers had to infer that swarm wants signals. External
observers had no entry point that wasn't also operational protocol.

## Signal
Human S443: "swarm wants to learn from all and teach all swarm this make it clear for the
rest who might interact with the swarm." First explicit signal about swarm's communication
stance toward outsiders (distinct from internal coordination).

## Resolution
docs/LEARN-AND-TEACH.md created. Explains: how signals travel, the compaction stack,
epistemic inventory, and the four-part contract (honesty/accessibility/self-challenge/
teaching edges). Teaches by naming unknowns, not only capabilities.

## Rule
Every system wanting external input must have an explicit "here is how to give us input and
here is what we do with it" entry point — not buried in protocol. Absence is itself a
barrier contradicting the stated learning goal.
