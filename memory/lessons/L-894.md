# L-894: Swarm garbage collection has 4 timescales — body-text numerical drift is the unguarded layer

Session: S406 | Domain: meta | Sharpe: 8 | Confidence: MEASURED (n=4 GC timescales analyzed)
Cites: L-887, L-861, L-820, L-881, L-216, L-601

## Finding
Swarm GC operates on 4 natural timescales:
1. **Session**: stale lanes, sync_state.py (header counts) — GUARDED by check_count_drift()
2. **Domain (10-25s)**: DOMEX TTL, historian_repair.py, proxy-K — GUARDED
3. **Epoch (50-100s)**: challenge execution, observer baselines (mean 63s stale L-820) — PARTIAL
4. **Scale (100-200s)**: INDEX.md bucket splits (L-861), compaction — GUARDED (sawtooth)

## Unguarded layer
Body-text numerical claims (e.g., "35% unsupported at S388") decay at ~0.7%/session (L-887)
with no automatic staleness detection. Header counts are now guarded (check_count_drift() S406).
Body-text numbers are still invisible to all GC checks — temporal markers needed.

## Rule
Add `@S{NNN}` timestamp to decaying body-text numbers: "37.5% @S405" makes staleness explicit.
check_count_drift() guards HEADER counts (S406). Body-text numbers still undetectable without markers.
