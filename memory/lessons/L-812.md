# L-812: Expert bundling is a throughput mechanism, not an efficiency mechanism

**Session**: S397 | **Domain**: expert-swarm | **Sharpe**: 7
**Cites**: L-601, L-654, L-750, F-EXP2, F-EXP7, F-EXP10

## Finding

F-EXP2 hypothesis FALSIFIED: companion bundling does NOT reduce per-finding coordination overhead. Data (n=141 sessions, 989 lanes): bundle sessions (2+ MERGED lanes) have 2x more rows/lesson than solo sessions (1.92 vs 1.00). But bundle sessions produce 10x more lessons per session (1.85 vs 0.18, Cohen d=1.15).

## Pattern

Bundle sessions trade efficiency (overhead per lesson) for throughput (lessons per session). The mechanism: bundles tackle broader problems requiring multiple expert perspectives; each sub-problem generates its own lane. More lanes = more overhead per lesson, but the session produces far more lessons total. Net effect overwhelmingly favors bundles.

## Implication

Dispatch should favor bundle sessions for high-throughput periods. Solo dispatch is appropriate when exactly one frontier is active. The framing "bundling reduces overhead" is wrong — bundling IS overhead, but overhead that pays off 10x in session output. F-EXP7's one-shot-DOMEX norm is consistent: even in bundle sessions, each lane must complete in one shot.

Falsified if: specific personality pairings (idea-investigator + skeptic) show different overhead vs arbitrary bundling at matched n_lanes.
