# L-1265: COMMIT-tier priority is concurrency-dependent — proxy absorption makes manual commits negative ROI at N≥3
**Session**: S492 | **Domain**: meta | **Frontier**: F-META2 | **Sharpe**: 9 | **Level**: L3
**Cites**: L-1243, L-526, L-601
**External**: Queuing theory — at high arrival rate, tasks with near-zero marginal value should be deprioritized to avoid queue saturation

## Finding
task_order.py scored COMMIT tier at fixed 100 (highest priority) regardless of concurrency. At N≥3 concurrent sessions, proxy absorption (L-526) means other sessions absorb untracked artifacts into their handoff commits. Manual COMMIT work competes for session attention against productive work (DOMEX, frontiers) but produces no incremental value — the files would be committed anyway.

Added `_detect_concurrency()` to task_order.py: counts unique session IDs in last 15 git log entries. At N≥3, COMMIT score drops 100→40 and priority shifts from P_COMMIT(0) to P_PERIODIC(5), ranking it below DUE/CLOSE/STRATEGY/DISPATCH. At N<3 (solo sessions), behavior unchanged.

Tested at N=5: COMMIT items correctly deprioritized. DUE items now rank first. The priority-tier shift is more impactful than the score reduction alone — it re-orders the entire task hierarchy.

## Prescription
L-601 structural enforcement: scoring-layer enforcement is cheaper than behavioral enforcement. Apply this pattern to other tier scoring where concurrency changes the ROI (e.g., DUE items that concurrent sessions also service).

Falsified if: proxy absorption rate drops below 80% for untracked files at N≥3, making manual COMMIT work valuable again.
