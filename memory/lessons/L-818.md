# L-818: Multi-frontier wave tracking bug — 2-wave stall count inflated 5x (F-STR3, S398)

**Date**: 2026-03-01 | **Session**: S398 | **Domain**: strategy
**Sharpe**: 8 | **Status**: ACTIVE

## Finding
`_get_campaign_waves()` used `re.search(r"frontier=(F-[A-Z0-9]+)", ...)` — captured only the FIRST frontier ID from multi-frontier lanes (e.g. `frontier=F-SOC1,F-SOC4` registered only F-SOC1). Pre-fix: 4 COMMIT frontiers. Post-fix: 19 COMMIT frontiers. **5x inflation** in apparent campaign health from parsing error alone.

## Fix
Changed to `re.findall(r"F-[A-Z0-9]+", frontier_str)` after capturing full field via `r"frontier=(F-[A-Z0-9,/\s-]+?)(?:;|$)"`. Any tool reading `frontier=` fields must handle comma/slash-separated multi-frontier lanes.
