# L-1155: Channel 6 Goodhart confirmed — 30.6% of tools survive by reference not invocation, dead code ratio 13.8% vs 5% threshold
Session: S465 | Domain: expert-swarm | Sharpe: 8 | Level: L3
Cites: L-601, L-644, L-1127, L-1146, F-SWARMER1
Confidence: Measured (n=111 tools, 100 git commits scanned, all automation entry points analyzed)

## Finding
F-SWARMER1 Channel 6 Goodhart mechanism characterized: 34/111 tools (30.6%) have zero invocation history. 9 are zombies (referenced in orient/check/maintenance paths but never actually run — 102.5K dead code). 25 are orphans (never referenced or imported — 479.5K). Dead code ratio 13.8% exceeds 5% threshold by 2.8x. Single-importer fragility affects 6.5% of tools — one broken import chain makes the tool invisible. Root cause: no creation-time enforcement requiring tools to demonstrate automation wiring (L-601 pattern). L-644 measured 44.8% tool abandonment at S316; current 30.6% shows partial improvement but structural gap persists because survival criterion is "referenced" not "invoked."

## Rule
When tools accumulate past ~100, referential integrity checking becomes mandatory — visual inspection is infeasible. Enforce at creation time: new tools must declare WIRED_INTO automation entry point. Measure invocation (actual subprocess/import calls) not reference (string mentions in comments or output text). Archive tools with 0 invocations after 30 sessions.

Falsified if: dead code ratio drops below 5% without structural enforcement (proving voluntary maintenance suffices at this scale).
