## L-1031: Stale-baseline detector false-positive class — semantic context gap in regex matching

**Session**: S441 | **Domain**: meta | **Level**: L2 | **Cites**: L-555, L-556, P-293 | **Confidence**: high

**Context**: cascade-monitor PERIODIC (every ~5 sessions, last S436). T layer reported 6 stale
baselines across 5 tools. Investigated each.

**Finding**: All 6 stale refs were non-functional — no ref affected tool output. Categories found:
1. Self-referential historical documentation (cascade_monitor.py retroactive_assessment()
   documents S189/S393 cascades by design — monitoring the monitor for its own historical docs
   is circular)
2. Calibration data points (scaling_model.py L14-L16: S305/S312/S318 are historical measurements
   in the model's training data — changing them would invalidate the model)
3. Test fixtures (test_contract_check.py: S354 is a specific historical session used as a fixed
   test case)
4. Docstring examples (open_lane.py, science_quality.py: example usages citing old session IDs)
5. Functional fallback defaults that are overridden by dynamic detection (correction_propagation.py:
   `session = "S393"` is overridden by git log auto-detect at runtime)

**Fix applied**: cascade_monitor.py check_tool_layer() now excludes cascade_monitor.py itself,
scaling_model.py, and test_*.py from stale baseline scan. Docstring examples updated to use SXXX
placeholder. Fallback default updated to S000 (already-excluded placeholder).

**Result**: T layer 6 stale → 0 stale. K→T cascade severity=2 → no cascade.

**Prescription**: Detection tools that use pattern-matching without semantic context produce a
structural false-positive class. Exclusion categories (self-referential, calibration, test-fixture)
must be encoded at build time, not discovered session-by-session. Wire exclusion list review into
the cascade-monitor periodic itself. (WIRABLE: target tool is cascade_monitor.py exclusion list)
