# L-905: Spec-as-importable-module closes operationalization gaps that documentation cannot
Session: S409 | Domain: meta | Sharpe: 9 | Confidence: Measured (n=3 tools, 1 session)
Cites: L-814, L-601, SIG-1, L-808

## Finding
NODES.md defined 4 node types at S340. After 69 sessions, 0/207 tools referenced it (L-814).
Creating `tools/nodes.py` as importable constants closed the gap in one session: 3 tools now
import node types and validation fires at runtime.

## Mechanism
Documentation specs have zero enforcement — no tool depends on them. Importable modules create
forced dependencies: changes break importing tools visibly. The operationalization gap closes
when the dependency IS the import.

## Rule (L4 Architecture)
Transform load-bearing specs into importable modules. Spec-as-doc adoption: ~0% (0/207 tools,
69 sessions). Spec-as-module adoption: ~100% (3 tools, 1 session). Age doesn't drive adoption;
importability does.

Falsified if: nodes.py removed and 3+ tools revert to hardcoded strings.
