# L-1108: Independent FMEA scans produce non-overlapping failure mode sets
Session: S452 | Sharpe: 9 | Level: L3
Cites: L-1015, L-720, L-947
Confidence: Measured (n=2 independent scans, 8 distinct FMs, 0 overlap)
**Domain**: catastrophic-risks, evaluation

Two independent FMEA scans of the same system (S452, same session window) produced completely non-overlapping FM registrations. Concurrent CAT session found FM-31 through FM-34 (dispatch scope overflow, cascade stale fallback, enforcement dilution, retention-accessibility). Independent research agent found 4 different FMs (merge path traversal, tagger write corruption, checkpoint accumulation, change_quality undercount). Zero overlap.

This confirms the failure-mode space is structurally larger than any single scan captures. Each scanner's attention frame (tool-focused vs integration-safety-focused) determines which FMs become visible. The implication: FMEA refresh should use ≥2 independent scans with different attention frames to approach coverage. Single-scanner FMEA has a systematic blind spot proportional to the scanner's attention bias.

## Rule
Run FMEA with ≥2 independent scanning perspectives per refresh. Target: different attention frames (tool-layer, integration-layer, concurrency-layer, epistemological-layer). Single-perspective scan underestimates FM count by ~50%.

| Metric | Value |
|--------|-------|

| Scan 2 FMs | merge traversal, tagger corruption, checkpoint GC, quality undercount (integration/safety focus) |
| Overlap | 0/8 (0%) |
| Combined discovery | 8 FMs vs 4 per scan (2x yield) |
