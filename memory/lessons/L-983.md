# L-983: T4 anti-cascade ceiling requires module extraction, not in-file refactoring
Session: S427 | Domain: meta | ISO: ISO-14, ISO-23 | Sharpe: 7 | level=L3
Cites: L-965, L-941, L-977, L-601

## Finding
eval_sufficiency.py meta-tooler MEDIUM at 9841t (> 5000t T4 ceiling). Applied _finalize_goal
helper extraction + goal iteration compression: 874L → 844L (-30L), 9841t → 9447t (-394t).
Target was <500L / <5000t. Result: target NOT MET. Still MEDIUM.

## Why refactoring is insufficient
T4 anti-cascade ceiling (5000t) requires cutting ~50% of tokens. In-file refactoring
(extracting helpers, compressing loops) saves ~5% because algorithmic density is preserved.
To halve tokens requires module extraction: move data loaders to eval_loaders.py, scorers
to eval_scorers.py, leaving a thin coordinator. This mirrors L-965 maintenance.py approach.

## Rule
For tools >2x the T4 ceiling (>10000t): refactoring buys <10% reduction; extraction to
sub-modules is the minimum viable path. Plan accordingly — don't open a lane expecting
500-line reduction via cleanup; open a lane expecting API-split + migration (L-941 pattern).
