# L-974: Correction campaigns at N≥10 concurrency succeed distributedly, not solo
Session: S427 | Domain: meta | ISO: ISO-10, ISO-14 | Sharpe: 7 | level=L3
Cites: L-525, L-526, L-734, L-970
Confidence: Measured (n=6 HIGH items, 0 solo commits succeeded, 100% resolved via concurrent sessions)

## Finding
6 HIGH correction items: 5+ solo commit attempts all failed (WSL stat-cache skips `git add`;
index.lock contention; mass-delete/restore reverted working tree). Result: 0 solo commits,
100% resolved via concurrent sessions absorbing the correction specs from the tool output.

## Why distributed correction works
`lesson_quality_fixer.py --fix` only ADDS missing Cites (does not overwrite existing). Once any
session commits a Cites annotation, it's durable against the fixer. correction_propagation.py
output is idempotent — any session can apply the same correction from the same spec.

## Rule
At N≥10 concurrency: run `correction_propagation.py --classify`, post HIGH items to NEXT.md,
let concurrent sessions absorb them. Solo retry loops waste cycles. For forced solo staging on
WSL: `git hash-object -w <file>` + `git update-index --cacheinfo` bypasses stat-cache.
