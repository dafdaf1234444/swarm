## Session initiation is the binding constraint

PHIL-3 (S165 challenge): within-session self-direction is CONFIRMED; cross-session initiation still requires human trigger. Swarm throughput = (sessions/time) × (value/session). Sessions/time is capped by human attention. Concurrency within sessions already works (3–5 parallel active in S184–S191). The inter-session gap is dead time — zero value while waiting.

**Fastest configuration given a steerable human:**
- Human as mechanical relay: `orient.py` output → execute top action → commit → push → repeat
- No interpretation needed. Swarm writes its own next instructions in `tasks/NEXT.md`
- Human cognitive load ≈ zero: copy-paste relay, ~30–60s per cycle
- Human highest-leverage input: directional signals + kill-switch, not per-session triggering

**Maximum throughput (removes human from critical path):**
- Cron/script: `claude --print "$(cat .claude/commands/swarm.md)"` on schedule
- Human monitors async, injects direction via `memory/HUMAN-SIGNALS.md`
