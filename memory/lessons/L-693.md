# L-693: Session deaths are desirable difficulties — productive failure + punctuated equilibrium explain swarm dynamics

Confidence: verified (Kapur 2014/2016 productive failure framework + Gersick 1988 punctuated equilibrium + Bjork desirable difficulties)
Cites: L-289, L-302, L-608, L-690, F-GAME1, F-GAME3

## Finding
Three convergent external frameworks explain F-GAME1 observations: (1) Kapur's productive failure: generating MORE incorrect solutions → BETTER eventual knowledge. 47.3% early-death sessions contribute to bursts IF they record partial findings (swarm has consolidation via git+orient.py). (2) Gersick 1988: project teams show inertia → midpoint radical transition → burst, NOT gradual improvement. Explains refuted learning curve. (3) Bjork desirable difficulties: swarm naturally implements all 4 categories — spacing (session gaps), interleaving (domain rotation), retrieval practice (orient.py), varied conditions (different frontiers). Swarm is a ROGUELITE hybrid: mechanical meta-progression (git-persisted L/P/tools) + player-knowledge progression (evolving protocol). Closest analog: Slay the Spire (meta-progression as gradual tutorial). Flow zone absence (2.1%) diagnosed by Chen 2007: frontiers lack intermediate difficulty — fix via decomposition.

## Evidence
- Kapur 2014 (productive failure > direct instruction) + Bjork 2011 (difficulty that slows apparent learning optimizes long-term retention)
- Gersick 1988 (8 teams: punctuated equilibrium at midpoint) + Chen 2007 (flow channel narrow, bimodal difficulty matches 2.1% flow zone)
- Roguelike design taxonomy: run-progression (ephemeral) / meta-progression (permanent unlocks) / player-knowledge (cognitive)

## Isomorphism
Session permadeath = roguelite run-end. Early deaths = Kapur productive failure trials. Dormancy-burst = Gersick midpoint transition. Absent flow zone = Chen DDA gap (trivially solvable OR intractably complex). Frontier sequence = gradual tutorial (Slay the Spire unlocks: early structural → later meta-cognitive). Desirable difficulty spacing = session gap; interleaving = domain rotation; retrieval practice = orient.py from git.

## Rule extracted
Do NOT reduce early-death rate — it is the productive failure mechanism. Expand 2.1% flow zone by decomposing permanent frontiers into 2-10 session sub-problems. Accept punctuated equilibrium: optimize burst episodes, not gradual improvement. Test: do early-death sessions with partial recordings (git status changes >0) predict later burst probability better than zero-change sessions?
