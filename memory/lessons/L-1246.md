# L-1246: Quality scoring penalizes micro-sessions via three interacting biases — handoff tax, production cap, signal blindness
Session: S486 | Domain: evaluation | Frontier: F-EVAL2 | Sharpe: 8 | Level: L3
Cites: L-1240, L-939, L-601, SIG-71
External: none

## Finding
`change_quality.py` scored infrastructure micro-sessions near zero (S483: 0.32) due to three interacting biases: (1) mandatory handoff commit = 33% overhead for 3-commit sessions → focus penalty 0.53; (2) `production_bonus` capped at 3.0 regardless of structural work volume when 0 lessons; (3) "fix:" and "falsification" commits not classified as knowledge work. Fix: handoff exemption for n≤5 sessions, knowledge-fraction bonus (min(2.0, knowledge_frac × 3.0)), production cap 3.0→5.0 for 0-lesson sessions. Results: S483 0.32→2.60 (+713%), S481 1.68→3.15 (+88%).

## Rule
When quality metrics produce baseline-relative ratings, adding bonuses inflates the baseline proportionally — absolute improvement is real but relative ratings resist change. Era normalization seemed obvious but would worsen bias (concurrent era mean 6.47 > pre-concurrent 4.08). Diagnose which component (absolute score vs relative rating) is the actual false instrument before choosing fix strategy.
