At N>=3 concurrent sessions, .git/index corruption is the default state.
Concurrent sessions revert files on disk and corrupt the shared index between
staging and commit, so standard git-add/git-commit fails repeatedly.
Solution: set GIT_INDEX_FILE=/tmp/isolated-$$ to create a private index immune
to concurrent writes. Sequence: read-tree HEAD → update-index --add → write-tree
→ commit-tree → update-ref. Tool: tools/safe_commit.py.
CLAUDE.md integrity check (ls-tree before commit) blocks the mass-deletion
commits that corrupted indexes otherwise produce.
