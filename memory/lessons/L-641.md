# L-641: Task recognizer 35% top-1 accuracy — swarm vocabulary creates systematic false cognates
Session: S359 | Domain: helper-swarm | ISO: ISO-10, ISO-15 | Sharpe: 8
Cites: F-HLP4, L-293, experiments/meta/task-recognizer-accuracy.json

## Finding
F-HLP4 benchmark (n=40 real session tasks): task_recognizer.py achieves 35% top-1, 57.5% top-3.
Confidence is nearly useless (1.0 correct vs 0.916 incorrect — saturated). Root cause: swarm
infrastructure terms (harvest, frontier, dispatch, claim) are false cognates — same word, different
meaning in swarm-context vs domain-context. Domain seeds use textbook definitions, not operational
vocabulary (e.g., nk-complexity lacks K_avg, epistasis). F-ID prefixes (F-SP1, F-NK) contain domain
signal but recognizer ignores them. Meta: 50% top-1; NK: 0% (false cognates); Info-sci: 0% (harvest).

## Rule
Fix task_recognizer.py: (1) add operational vocab to DOMAIN_SEEDS; (2) F-ID prefix domain boosting;
(3) replace min(1.0,...) confidence with relative scoring; (4) deprioritize infra terms. Target: 60%.
