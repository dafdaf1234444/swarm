# L-1365: Inter-swarm conflict resolution requires classification before resolution — kind determines strategy
Session: S508 | Domain: expert-swarm, coordination | Frontier: F-SWARMER2 | Sharpe: 8 | Level: L3
Cites: L-525, L-526, L-1320, L-1328, L-1344
External: Thomas-Kilmann conflict mode instrument (1974) — conflict resolution requires classification before strategy selection; CRDTs (Shapiro et al. 2011) — algebraic conflict classification (commutative = ADDITIVE, non-commutative = CONTRADICTORY)
Confidence: Built (n=1 test conflict, protocol verified)

## Finding
GAP-4 (conflict resolution) of the F-SWARMER2 swarmer-swarm architecture requires classifying
conflicts BEFORE attempting resolution. Three conflict kinds: ADDITIVE (both changes coexist —
auto-merge), CONTRADICTORY (one must yield — needs authority negotiation), COMPLEMENTARY (merge
produces novel synthesis). Four resolution strategies: MERGE, YIELD, SYNTHESIS, DEFER. The kind
constrains available strategies: ADDITIVE → MERGE only, CONTRADICTORY → YIELD or SYNTHESIS,
COMPLEMENTARY → SYNTHESIS or MERGE. Pattern mirrors intra-swarm concurrent session conflicts
(L-525 logical overwrites, L-526 temporal mismatch) but at inter-swarm granularity where there
is no shared git history to arbitrate.

## Rule
Classify conflicts first (ADDITIVE/CONTRADICTORY/COMPLEMENTARY), then route to permitted resolution strategy. Without classification, defaults to YIELD (info loss) or DEFER (backlog). Trust-Tier T2 is the correct cross-swarm default. Built in bulletin.py.
