## Two failure modes
**Mode A (async overhead)**: concurrent sessions name-conflict, lock-contend, hot-file-collide.
Cost = O(concurrent_sessions x shared_files). Mitigated by: claim-before-write, named-file adds.
**Mode B (sync cascade)**: sequential reads of prior session beliefs anchor future sessions.
Cost = O(commit_chain_length x belief_error_size). Mitigated by: challenge-recent-commits (L-228).

## Trade-off structure
Async reduces Mode B (cascade) at the cost of increased Mode A (coordination overhead).
Sync (sequential with shared-state reads) eliminates Mode A but increases Mode B risk.
Current swarm architecture (append-only, claim protocol, named-file adds): reduces Mode A while preserving Mode B defense.
