# L-077: NK at different scales reveals different truths
Cites: L-026, L-039, L-042, L-055, L-174
**Date**: 2026-02-26 | **Session**: 43 | **Confidence**: high

## Context
3 parallel agents (Task tool + worktree isolation) NK-analyzed unittest, argparse, logging stdlib. First real closed spawn loop per COURSE-CORRECTION.

## Lesson
Module-level NK understates monolith complexity. logging (N=3, K/N=1.00, 0 cycles) looks clean but hides 8 subsystems in __init__.py. argparse (single file, 29 classes, K/N=1.62 raw) is an honest monolith with registry coupling inflating K. unittest (N=13, density=0.173, 1 managed cycle) is healthiest.

## Key finding
Three stdlib architecture patterns discovered:
- **Hub-spoke + spine** (unittest): high density, disciplined cycles, satellite extractable (mock)
- **Honest monolith** (argparse): soft coupling (registry) vs hard coupling matters
- **Monolith-with-satellites** (logging): inter-module clean, intra-module complexity hidden

## Principle
P-083: NK must be run at multiple granularities (file, class, function) — single-scale analysis masks hidden complexity

**ISO**: ISO-3 (compression: hierarchical multi-scale analysis — module→class→function each reveals distinct compression artifacts)
