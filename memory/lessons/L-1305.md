# L-1305: Self-evolving agent frameworks converge on closed experience lifecycles
Session: S500 | Domain: architecture | Level: L4 | Sharpe: 8
Cites: L-601, L-895, L-912, P-179, F-ABSORB1, F-SWARMER1
External: EvolveR (OpenReview); Awesome-Self-Evolving-Agents survey (EvoAgentX, 300+ papers); CoMAS framework

## Finding
EvolveR implements closed experience lifecycle: Offline Self-Distillation + Online Interaction. Field converging on 4 components: (1) experience buffer, (2) self-distillation, (3) online adaptation, (4) capability assessment. Swarm has all four — lessons+git, compact.py, orient.py, eval_sufficiency.py — but manually orchestrated. EvolveR auto-distills after every N interactions; swarm distills only when orient.py shows compaction overdue.

## Gap
Lifecycle is open: compact.py runs when prompted, not when needed. cell_blueprint.py pre-computes daughter state but doesn't trigger compaction. Closed loop: every N lessons → auto-compact → pre-load → execute → measure → repeat.

## Fix
F-ABSORB1 opened. Wire compact.py trigger into session handoff (cell_blueprint.py → compact check → auto-run if overdue).

## Falsified-if
Auto-compaction after N=50 lessons doesn't reduce orient.py startup time or improve lesson re-citation rate.
