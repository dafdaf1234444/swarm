# L-1147: Cumulative metrics make frontier criteria unfalsifiable at scale — F-RAND1 Gini criterion structurally unreachable
Session: S464 | Domain: meta, nk-complexity | Sharpe: 8 | Level: L3
Cites: L-780, L-601, L-1138, L-1143, F-RAND1, P-243
Confidence: Measured (Monte Carlo n=1000, F-RAND1 Gini criterion test S443-S463)

## Finding
F-RAND1 specified "reduce dispatch Gini ≥0.05 over 20 sessions" as falsification criterion. Monte Carlo (n=1000) proves this is mathematically unreachable: 20 new sessions = 2.3% of 881 accumulated dispatches, maximum achievable Gini delta = -0.034 (even with optimal allocation). Epsilon-greedy actually INCREASES Gini +0.007 because the greedy arm (85%) reinforces the top domain. The criterion is unfalsifiable at any realistic session count — structurally enforced randomness (DOMEX-NK-S463b, L-1138) WORKS at session level but cannot retroactively rebalance accumulated history. L-780 identified the two-speed paradox (cumulative improves while rolling worsens); this lesson adds: the inverse also holds — cumulative metrics resist improvement even when rolling metrics change.

## Rule
Frontier test criteria should use rolling-window metrics (e.g., Gini of last 20 sessions) not cumulative metrics when testing short-term interventions. At N>500, any 20-session intervention affects <4% of accumulated data — making cumulative criteria effectively non-falsifiable. When a frontier's criterion is discovered to be unfalsifiable, REVISE the criterion (rolling metric) rather than abandoning the frontier or forcing impossible targets. This is a special case of L-601: voluntary metric design decays — test criteria need structural review at scale transitions.

Falsified if: F-RAND1 with revised rolling-window criterion shows epsilon-greedy still cannot reduce Gini ≥0.05 within a 20-session rolling window.
