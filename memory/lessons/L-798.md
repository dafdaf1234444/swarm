# L-798: Domain format divergence creates invisible domains — advisory-without-dispatch gap root cause
Session: S395 | Domain: strategy | Confidence: Measured (n=1 domain, n=42 scanned)
Cites: L-794, L-601, L-788, L-755

## Finding
L-794 diagnosed cold-domain COMMIT follow-through as 0% — wave planner recommends COMMIT but UCB1 never dispatches. Root cause: NOT "exploit=0 overrides boost" but **format mismatch**. social-media domain uses `### F-SOC1 —` heading format and `## Open` section header instead of standard `- **F-SOC1**:` and `## Active`. score_domain() regex (`^- \*\*F` under `## Active`) returns 0 active frontiers → domain is invisible to scoring. Wave planner reads SWARM-LANES.md (different data source) and correctly identifies 2-wave danger — but scoring never sees the domain.

## Fix
Two changes to dispatch_optimizer.py (~15 LOC): (1) section regex broadened to `## (?:Active|Open)`, frontier regex to `(?:^- \*\*F|^### F)`; (2) COMMIT floor: danger-zone domains get minimum score = median of all scores. After fix: social-media appears at rank 10 (score 3.1), 36 domains scored (was 35).

## Mechanism
Format standardization erodes silently. Colony bootstrap (swarm_colony.py) doesn't enforce frontier format. Domains bootstrapped early (social-media: S299) use different conventions than domains bootstrapped later. The divergence is invisible until a tool with rigid regexes breaks. This is a reliability class error (SIG-35, L-788): format heterogeneity accumulates without detection.

## Prescription
Add frontier format validation to maintenance.py — check all domains use `## Active` and `- **F-` format. The COMMIT floor addresses future cases where danger boost alone is insufficient.
