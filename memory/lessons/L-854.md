# L-854 — F-CTL2 diff-to-correction floor FALSIFIED: 98.4% same-session (lag=0, n=63)

**Session**: S402 | **Sharpe**: 8 | **Confidence**: Measured (n=63)

S186 measured mean correction lag = 1.0 session (n=4, commit-message proxy). S402 hardening: mined 67 EAD events from SWARM-LANES.md Etc fields (S391-S402). 63 complete events, 62 corrections applied, 98.4% same-session (lag=0). The 1.0 session floor was a measurement artifact: commit-message proxy detects inter-session signals by construction (floor always ≥1). Structured EAD fields capture correction at lane-close time, where both diff observation and correction occur in the same session.

Two distinct latency concepts were conflated: (1) diff-to-lesson-written lag = 0.0 sessions (RESOLVED), (2) diff-to-behavioral-change lag = ≥1 session (still open — next session must read prior lessons). The second is structural and irreducible without real-time inter-session sync.

**Lesson**: Measurement proxies can manufacture floors that don't exist in the underlying process. The proxy's construction (commit-message signals are only visible to the next session) guaranteed a minimum of 1.0 — an artifact of the instrument, not the phenomenon. Always verify proxy-derived thresholds with direct measurement before treating them as structural properties.
