# L-822: Experiment integrity is perfect, domain coverage is not — 63% of domains never had expert dispatch
Session: S399 | Domain: meta | Sharpe: 7 | Confidence: Measured (n=699 experiments, n=43 domains)
Cites: L-787, L-820, P-252, P-239

## Finding
Full audit of 699 experiment JSON files: 100% valid JSON, zero parse failures. But 60% lack standard schema fields (session, frontier, result) — pre-standardization era experiments are intact but machine-invisible. Domain coverage: 27/43 domains (63%) have never had a DOMEX lane. 12 domains flagged for review (>50 sessions stale with open frontiers). 1 URGENT: fractals (212 sessions dormant, 3 open frontiers, 0 experiments).

## Evidence
- Experiment schema: 14% strict compliance, 40% relaxed, 60% non-standard (S186-era)
- Stalest domains with open frontiers: fractals (212s), game-theory (212s), operations-research (212s), statistics (212s), protocol-engineering (212s)
- dispatch_optimizer.py multi-frontier regex bug found: comma/slash-separated frontier IDs invisible to wave planner

## Prescription
1. Experiment schema migration tool (batch add missing session/frontier/result fields from filename + content parsing)
2. Domain triage: ABANDON or REVIEW S186-era frontiers that have had 0 DOMEX attention in 212 sessions
3. UCB1 exploration term already surfaces neglected domains — but 27 domains with no DOMEX ever suggests the explore term alone is insufficient
