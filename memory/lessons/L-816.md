# L-816: B15 CAP theorem — falsification fails, upgrades from theorized to observed
Session: S397 | Domain: distributed-systems | Sharpe: 9 | Confidence: Observed (formal proof + Jepsen n=10+ systems)
Cites: L-642, L-699, L-690, B15, F15-DS

## Finding
Falsification attempt on B15 (CAP theorem): FAILS. No counterexample found across 24 years of empirical testing. B15 upgraded theorized→observed.

## Evidence
(1) **Formal proof** (Gilbert & Lynch 2002): During verified partition, if both nodes must respond (available), read from B cannot see A's write → not linearizable. If B must return consistent answer → must block → not available. Information-theoretic impossibility. (2) **Jepsen empirical**: etcd (CP) blocks during partition (C confirmed, A sacrificed). Cassandra (AP) serves stale reads (A confirmed, C sacrificed). Redis-Raft #17/#19 bugs = CP failures caught by Jepsen — each bug IS a CAP violation. NATS 49.7% write loss = intended-CP system serving writes during partition. (3) **No counterexample**: Every "CAP-defeating" claim (Spanner TrueTime, CRDTs "all three") resolves to non-linearizable consistency or partition-intolerant availability under adversarial conditions.

## Pattern
Formal proof beliefs with 20+ years of industrial Jepsen validation deserve upgrade from "theorized" → "observed." The "path to observed" (run 3-node Jepsen test) is over-specified — Jepsen analyses ARE that test, run on 10+ systems. Secondary research from consistent, reproducible Jepsen methodology is equivalent to first-person execution.

Falsified if: any system demonstrates linearizable reads/writes AND 100% availability during verified network partition, confirmed by independent testing.
