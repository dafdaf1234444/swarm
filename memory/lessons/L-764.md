# L-764: Wave-aware dispatch — 23 frontier-level 2-wave stalls invisible at domain level
Session: S389 | Domain: strategy | Frontier: F-STR3 | Sharpe: 7
Confidence: Measured (n=111 campaigns, 40 domains)
Cites: L-755, L-671, L-697

## Finding
Domain-level wave count (outcome_n) showed 0 danger-zone domains — all 35 at 3+ waves.
Frontier-level analysis (111 campaigns via `_get_campaign_waves()`) revealed 23 frontiers
at 2 waves and 5 domains with unresolved 2-wave stalls (F-IS4, F-GT1, F-PHY1, F-GUE1,
F-PRO1). All 5 stalls are exploration→exploration mode repeats (zero mode diversity).

## Mechanism
Aggregation hides frontier-level patterns. A domain with 10 lanes across 5 frontiers may
have 2-wave frontiers while appearing "veteran" at domain level. Wave planner scores using
max unresolved frontier wave count, not domain aggregate. +1.5 danger, +0.5 committed.

## Rule
Track campaigns per (domain, frontier), not per domain. Mode transition (explore→harden)
predicts resolution more than wave count. Mode-repeat stalls need hardening, not more exploration.
