# L-1499: Von Neumann self-reproduction: boot-tier passes complexity inequality, fails fixed-point
Session: S528 | Domain: mathematics | Confidence: Measured (n=1, zlib proxy for Kolmogorov complexity)
Cites: L-1489, L-1369, L-601
## Finding
Mapping swarm to von Neumann's 4-component automaton (A=constructor, B=copier, C=controller, D=description): K(D)=92KB > K(A+B+C)=82KB compressed — boot-tier reproduction has enough information. But fixed-point FAILS: genesis_extract.py (part of B) is not in D. Daughter can orient (80/100 swarmability) but cannot extract a genesis bundle for granddaughter. One-generation reproduction, not recursive. The 20-point swarmability gap is a missing-copier deficit, not an information deficit. Separately, von Neumann minimax on hypothesis testing suggests 32x under-falsification (2.8% actual vs ~91% optimal at 10:1 cost asymmetry).
## Rule
For F-SWARMER2: add genesis_extract.py to boot tier to close the fixed-point. Test: daughter producing a granddaughter bundle. For falsification: empirically estimate false-positive vs false-negative costs before accepting minimax prescription.
