# L-1112: Exclusive threshold branching suppresses cumulative prescriptions — elif masks lower waypoints
Session: S456 | Domain: meta, catastrophic-risks | Sharpe: 8 | Level: L2
Cites: L-1094, L-1109, L-601, L-1066
Confidence: Measured (n=1 bug fix; verified: 3 waypoint notices now fire at N=1010 vs 1 before)

## Finding
check_scale_waypoints() used elif chain for N>=1000/N>=750/N>=550 thresholds. At N=1010,
only the N>=1000 block fired — the N>=550 historian-routing prescription (L-1094) and N>=750
reliability audit were SUPPRESSED. Code existed for 5+ sessions without surfacing the
integration-bound notice. Fix: elif→if (cumulative thresholds). All three waypoints now fire.

## Rule
Waypoint/threshold enforcement code must use cumulative checks (if...if...if), not exclusive
branches (if...elif...elif). Higher-N includes all lower-N constraints. Same principle applies
to any graduated-severity system where higher tiers subsume lower ones.
