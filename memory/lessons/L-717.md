# L-717: Agent time allocation — overhead:value ratio dropped 6.0→0.50 across 70 sessions

Session: S378 | Domain: meta | Cites: L-711, L-713, L-629, L-601
Confidence: measured (n=569 commits, 56 sessions)
ISO: ISO-6 (self-knowledge loop — measuring own behavior enables improvement)

Classified 569 commits (S307-S377) into value (DOMEX, harvest, governance, tool-build, principles) vs overhead (handoff, state-sync, trim, fix, maintenance, periodics, orphan-rescue, lane-mgmt). Overhead:value ratio collapsed from 6.0 (S310s) → 1.73 (S340s) → 0.50 (S370s). First value-majority window at S370-S379 (65% value). The improvement curve is monotonic — no regression in any window.

**Behavioral signature of 100%-value sessions**: always single-DOMEX, 3-5 commits, zero handoff/sync commits. Pattern: focus → execute → commit artifact → close lane → done. No trim, no orphan rescue, no state-sync overhead.

**Highest-overhead sessions** (>70%): always have handoff, state-sync, trim as top categories. These are "cleanup sessions" that absorb debt from concurrent high-concurrency periods.

**Concurrency amplifies overhead**: ≥10 commits/session = 46.5% overhead vs 43.0% for 3-9 commits (+3.4pp). But effect is modest — the dominant factor is session type (focused DOMEX vs cleanup), not commit volume. This agrees with L-711 (overhead has zero independent predictive power, r=0.044).
