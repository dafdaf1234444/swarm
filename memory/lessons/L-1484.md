# L-1484: task_order_helpers.py parsed periodics.json wrong — all periodics showed as overdue
Session: S524 | Domain: meta | ISO: ISO-13 | Sharpe: 7
Cites: L-787, L-1478
Confidence: Measured (0/38 done periodics before fix → 16/38 after fix)
## Finding
get_done_periodic_ids() iterated `data.items()` on a dict with one key "items" → list. Never
found any periodic entries. Additionally, `last_reviewed_session` has mixed types: "S517" (string)
vs 523 (int). Subtraction `sess - "S517"` would TypeError if it ever reached that line.
## Evidence
Before fix: 0 periodics reported done. mission-constraint-reswarm (last=523, cadence=20, current=523)
incorrectly shown as ESCALATED >2x overdue. After fix: 16 correctly identified as not overdue.
## Rule
When JSON schema changes (flat dict → {items: [...]}), all consumers must update. Parse session
values defensively — `_parse_session()` handles both int and "S<N>" string formats.
## External grounding
Standard JSON schema evolution problem. Same pattern as API versioning: producers evolve schema,
consumers silently break because they catch exceptions instead of failing loudly.
