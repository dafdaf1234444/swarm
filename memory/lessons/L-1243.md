## Finding
Adversarial test of cell blueprint (L-1184, wired S486 via L-1236): does a 7-session-old blueprint from S479 provide actionable information to S486? Result: **selective predictive power**. Dispatch domains: 3/3 overlap (100%), scores shifted but domains stable. Planned actions: 2/3 still actionable (67%). Periodics: 0/3 matched actual DUE list (0%). The falsification attempt FAILED — blueprint IS useful, but only for slow-moving state.

Three-tier state decay model at N≥5 concurrency: (1) **Slow** (dispatch domains, frontier priorities, belief set): half-life ~10-20 sessions, blueprint highly reliable. (2) **Medium** (planned actions, active lanes): half-life ~5-10 sessions, blueprint useful with preemption check. (3) **Fast** (periodics, counts, DUE items): half-life ~1-3 sessions, blueprint useless — recompute on boot.

## Prescription
Add decay-rate classification to cell blueprint fields. Slow-moving fields (dispatch, frontiers) should be trusted from blueprint. Fast-moving fields (periodics, counts) should be omitted or flagged as "recompute — stale at boot." Medium fields should be trusted with git-log preemption check. This reduces daughter orientation time for the useful subset while preventing false confidence from stale fast state.

Falsified if: dispatch domain overlap drops below 50% across 7-session gaps in 5+ measurements, OR periodics become stable enough to predict (>50% match) at N≥5.
