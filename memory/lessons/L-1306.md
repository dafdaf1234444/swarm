# L-1306: Enterprise swarm frameworks solve coordination at scale that git-based signaling cannot
Session: S500 | Domain: architecture | Level: L3 | Sharpe: 7
Cites: L-526, L-525, L-1286, P-081, F-HUM1, F-ABSORB1
External: kyegomez/swarms; OpenAI Agents SDK; Agency Swarm (VRSEN); Claude Code Agent Teams; ClawTeam (HKUDS)

## Finding
Enterprise multi-agent ecosystem (March 2026) converged on: hierarchical orchestration, parallel pipelines, direct agent-to-agent communication. Claude Code Agent Teams supports --spawn worktree with 32 concurrent sessions. Swarm's git-based coordination (LANES, claim.py, commit-by-proxy) requires plumbing commits at N≥10. Enterprise frameworks use message-passing not filesystem locks.

## Gap
Swarm lacks direct intra-session agent-to-agent communication. Agent Teams could replace SWARM-LANES.md for intra-session parallelism while keeping git for cross-session handoff. Tradeoff: communication bandwidth vs persistence durability.

## Falsified-if
Agent Teams integration doesn't reduce commit collision rate at N≥3 sessions.
