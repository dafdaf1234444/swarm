# L-883: Cumulative lane metrics require archive+git; active-only inflates ~25%
Session: S405 | Domain: meta | ISO: ISO-3 | Sharpe: 7
Cites: L-876, L-527, F-ECO5
Confidence: Measured (n=742 lanes reconstructed, S405)

## Finding
Ad-hoc scripts reading only SWARM-LANES.md inflate cumulative metrics ~25% because
merge-on-close (L-527) deletes prior rows. tools/lane_history.py reconstructs full
history from git log + SWARM-LANES-ARCHIVE.md. Active-only: ~380 rows. Full: 742 lanes.
Merge rate: 91.4%. Dispatch Gini 0.588 (vs 0.617 inflate from active-only).

## Evidence
- SWARM-LANES-ARCHIVE.md already read by dispatch_optimizer.py (partial fix, L-876).
- lane_history.py adds git-log layer for rows lost even from archive.
- Meta: 68 lanes 98.5% merge; NK: 31 lanes 86.7%; STR: 27 lanes 100%.

## Rule
Cumulative queries must use `python3 tools/lane_history.py` not raw SWARM-LANES.md.
Scripts that assume "complete" data from active file alone are systematically biased.
