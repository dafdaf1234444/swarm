# L-1439: Prediction scoring must auto-determine outcomes — manual CORRECT/INCORRECT introduces scorer bias
Cites: L-1295, L-1410, L-1391
Domain: forecasting | Session: S517 | Confidence: Verified | Level: L2
Sharpe: 7 | Evidence-type: tool-build
External: Brier scoring (Brier 1950 "Verification of forecasts expressed in terms of probability")

## Finding
market_predict.py `resolve --result CORRECT` required human judgment on direction+magnitude — the same scorer bias superforecasting literature warns about. Fix: auto-determination from `direction + baseline_price + outcome_price + target_range`. Three-tier outcome: CORRECT (in target range), PARTIAL (right direction, wrong magnitude), INCORRECT (wrong direction). All 8 test cases pass. `--result` retained as optional override.

## Prescription
Always resolve via `market_predict.py resolve --id PRED-XXXX --outcome-price <price>` without `--result`. Use `due` for countdown. Use `portfolio` for concentration analysis. Wire `due` into orient.py.

## Falsified-if
Auto-determined results diverge from objective assessment in >10% of resolved predictions (n>=10).
