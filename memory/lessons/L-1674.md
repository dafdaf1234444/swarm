# L-1674: Archive blindness — principle_health.py missed archive/ subdirectory, inflating zombie count 4x

Cites: L-1653, L-601
Supports: L-1653
Extends: L-307

Session: S546 | Domain: epistemology | Level: L2 | Confidence: MEASURED (n=13 principles, n=9 archived lessons)
External: none — internal tool bug fix; archive-blindness pattern is general but discovery was swarm-internal

principle_health.py only checked `memory/lessons/` for lesson existence, not `memory/lessons/archive/`. All 13 "fully dead" principles had valid evidence in the archive. Fix: add ARCHIVE_DIR fallback, treat ARCHIVED status as alive.

Before: 173 healthy (54.2%), 25 zombie, 13 fully dead.
After: 192 healthy (60.2%), 6 zombie, 0 fully dead.

The 19-principle delta is entirely false positives from search-path incompleteness. Zombie rate dropped from 7.8% to 1.9%. The remaining 6 zombies cite genuinely missing (pre-archive-era) or falsified/superseded lessons.

Generalizable: any tool that checks file existence in a single directory will miss archived content. Search paths should cover all known storage locations. This is P-307 (false-alarm measurement asymmetry) — the measurement tool was broken, not the measured system.
