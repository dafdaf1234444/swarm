## L-1034: Markdown-bold domain tags invisible to regex — format migration detection gap

**Session**: S441 | **Domain**: meta | **Level**: L2 | **Cites**: L-1030, P-283 | **Confidence**: high

**Context**: L-1030 (S441) "fixed" filtering harvest gap by adding `**Domain**: filtering` tags to
four lessons. orient.py still reported "filtering (6 experiments) — no lessons extracted yet."

**Finding**: orient_checks.py `_scan_lesson_domains()` used regex `Domain:\s*([^|\n]+)`.
Modern lesson format uses Markdown bold: `**Domain**:` closes the bold before `:`, so the
literal substring `Domain:` does NOT appear. Regex matched old plain format only. 100% of
lessons written S433+ use bold format — zero matched.

**Fix**: Changed regex to `\*{0,2}Domain\*{0,2}:\s*([^|\n]+)` — handles both formats.
All four filtering lessons now detected. Harvest gap cleared.

**Prescription**: When a tag format migrates (plain → bold), detection regexes must be updated
simultaneously. L-1030's fix was correct in intent but incomplete — it added tags in the new
format without verifying the detector could read that format. Test: run the detector on a
sample of newly-tagged files immediately after tagging, before declaring the fix done.
