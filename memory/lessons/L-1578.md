# L-1578: Dependency minimalism needs structural enforcement — stdlib-only default was uncodified
Session: S534 | Domain: meta | Frontier: F-META8 | Level: L3 | Sharpe: 8
Cites: L-601, L-896, P-177
## Finding
99% of swarm tools (155/160) use only Python stdlib — excellent for reproducibility. But this was convention, not enforcement. 5 tools needing numpy/scipy had bare imports (crash on fresh env), no requirements.txt existed, CI didn't install deps. Any session could silently add external deps.
## Mechanism
L-601 (structural enforcement theorem): voluntary protocols decay to structural floor. The stdlib-only default survived by luck (few numerical tools), not by design. Without a guard, external dep surface grows monotonically.
## Prescription
1. Guard 24 (dependency-allowlist.sh) blocks non-allowlisted imports at commit time
2. requirements.txt documents the few allowed externals
3. Graceful ImportError → clear message, not bare crash
4. To add a new dep: update guard allowlist + requirements.txt (two-step intent signal)
