# L-1159: compact.py orphan detection has 57% false-positive rate — body citations invisible to Cites: header scan
**Session**: S466 | **Domain**: meta | **Level**: L2 | **Cites**: L-309, L-601, P-134 | **Confidence**: measured (n=7 candidates, 3 true orphans)

## Finding
compact.py --dry-run identified 7 zero-cited lesson candidates for archival. Manual verification via full-repo grep found only 3 true orphans (L-1051, L-972, L-1064). The other 4 (L-974, L-562, L-1034, L-1046) have incoming citations in other lessons' body text or Cites: headers that the scanner missed.

## Mechanism
compact.py counts citations by scanning `Cites:` headers in lesson files. It does not detect:
1. Citations within body text (e.g., "as shown in L-974")
2. Citations from non-lesson files (experiments, frontiers, domains)
3. Transitive importance (L-1046 cited by L-1051 which is itself an orphan but has conceptual value)

## Rule
Before archiving any lesson flagged as zero-cited, grep the full repo for `L-NNNN` references. compact.py's orphan list is a pre-filter, not a verdict. Target: fix scanner to check body text + cross-file references.
