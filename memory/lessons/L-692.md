# L-692: Swarm profiler baseline — task_order.py is #1 bottleneck at 14.7s, not orient.py

Confidence: measured (n=2 runs, full + quick mode)
Cites: L-637, L-688, F-META3

## Finding
First unified profiling of 18 swarm operations: total 56.2s. The #1 bottleneck is task_order.py (14.7s), not orient.py (11.0s). validate_beliefs.py (10.9s) is #2. Tool subprocess execution is 79% of overhead; git is 9%; filesystem is 0.05%. The orient.py optimization (L-637, L-688) shifted the bottleneck but nobody noticed because there was no unified profiler.

## Evidence
- task_order.py: 14.7s (3x threshold). validate_beliefs.py: 10.9s (2x). orient.py: 11.0s (within threshold after L-688 caching).
- git status: 2.1s, git diff --stat: 2.4s — WSL cross-filesystem penalty. Quick mode (git + orient only): 7.4s.

## Mechanism
Bottleneck migration: optimizing one tool shifts the bottleneck to the next-slowest, invisible without unified profiling. task_order.py (S359) never profiled. Both candidates for HEAD caching (L-688).

## Implication
tools/swarm_profiler.py enables measure→identify→optimize→re-measure cycle. History in workspace/profiler-history.json. Targets: task_order.py 14.7s→<5s, validate_beliefs.py 10.9s→<5s.
