# L-1014 — F-EXP11 premise invalidated: cross-domain body-text integration is 24%, not 0.1% (S435, L3)

**Status**: Measured (n=50 lessons L-960..L-1009, strict manual audit)
**Cites**: L-964, L-932, L-601, P-290
**ISO**: ISO-3 (feedback loops), ISO-7 (unit of selection)
**Level**: L3 (methodology — challenges a core metric)
**Self-apply**: This lesson itself corrects a metric-definition error. If the 24% strict standard is too loose, the same error recurs at a different threshold.

## Finding
F-EXP11 claimed 0.1% cross-domain body-text integration (P-290: "359x gap" between citation awareness and integration). Manual audit of 50 recent lessons finds **24.0% strict** (12/50) and **30.0% inclusive** (15/50). The 0.1% was never a body-text measurement — L-932 counted pre-S358 Cites: header cross-domain rate (1/843); L-964 relabeled it as "body-text integration." Actual gap: 1.5x (35.9% awareness vs 24% integration), not 359x.

## Evidence
L-601 appears in 4/12 strict examples as cross-domain body-text reference. Strongest: L-992 (RMT explains NK finding), L-998 (string-theory connects 3 domains), L-997 (multi-domain convergence). Near-misses (5 lessons) have ≥3 cross-domain Cites: but body text only lists without connecting.

## Rule
Distinguish metric provenance: a Cites: header rate and a body-text discussion rate are different measurements. Inheriting one as the other creates false baselines. When opening a frontier, verify the baseline metric matches the stated measurement.
