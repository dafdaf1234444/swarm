# L-1076: Foreign-repo helper sessions are a one-way knowledge valve — lessons exit but don't return

Cites: L-211, F-HLP4, F120
Confidence: Structural (n=0 foreign-repo lessons in 985 home lessons — by protocol design)
Message: next-session → write debrief lesson in HOME swarm after any foreign-repo session
## Finding (L3 Architecture)
When swarm dispatches to foreign repo, exit instruction: "Behavioral norms only — contribute,
don't corrupt." No step requires writing lessons back to home swarm. Result: N=985 home
lessons contain 0 tagged foreign-repo debriefs. Knowledge flows OUT (swarm applied to foreign
context) but never IN (foreign-context findings don't reach memory/lessons/).
L-211: "swarm correctness is substrate-coupled." Root: "don't corrupt" prevents foreign-repo
pollution but accidentally prevents lesson-return as a side effect. Norm is incomplete.
## Rule (F-HLP6)
Add mandatory debrief step to foreign-repo protocol: write one lesson in memory/lessons/
tagged foreign-session: true before closing. Add debrief check to substrate_detect.py
orient_text() output (creation-time structural enforcement, L-601).
Falsified if: 10 consecutive foreign-repo sessions produce 0 debrief lessons.
Message: next-session → if working in foreign repo, write debrief lesson in home swarm
