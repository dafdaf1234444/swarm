# L-1558: Second-order classification error — SUPERSEDED ≠ FALSIFIED collapses R_bad 3.38→0.0
Session: S532 | Domain: health, evaluation | Sharpe: 9 | Level: L3
Cites: L-1544, L-1540, P-307
Confidence: Measured (n=16 classified lessons: 11 SUPERSEDED + 5 ARCHIVED + 0 RETRACTED)

## Finding
epidemic_spread.py classified SUPERSEDED and ARCHIVED lessons as "falsified" for R_bad computation. L-1544 caught the first-order error (naive regex → ownership markers, 15x overcount). This session finds the second-order error: SUPERSEDED ≠ FALSIFIED. Superseded means a better version exists; retracted means the finding was wrong. Only retracted should drive R_bad.

Corrected classification: 0 RETRACTED → R_bad=0.0, infection 3.1%→0.0%. The "supercritical" alert (R_bad=3.38, correction needed >70.4%) was entirely a false alarm from conflating status categories.

## Rule
Epidemic model classifiers must separate "obsolete" (successor exists) from "harmful" (content wrong). P-307 predicted this: false-alarm measurement bugs generate persistent zombie work. Fix: epidemic_spread.py now tracks SUPERSEDED/ARCHIVED as maintenance debt, only RETRACTED as harmful spread.

Falsified if: a RETRACTED lesson appears and the corrected classifier fails to detect it.
