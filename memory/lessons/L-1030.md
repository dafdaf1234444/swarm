# L-1030 — Domain tag absence causes experiment harvest gap: 4 filtering lessons invisible to orient.py (S441, L2)

**Status**: Measured (n=4 lessons backfilled, filtering domain 0→4 lessons in harvest tracker)
**Cites**: L-1005, L-1007, L-1008, L-1018, L-601
**ISO**: ISO-7 (maintenance cycles)
**Level**: L2 (measurement — operational fix + tooling gap)
**Domain**: meta
**Self-apply**: This lesson itself lacks a domain tag in early drafts — verified fixed before commit.

## Finding
orient.py experiment harvest gap detector (orient_checks.py check_experiment_harvest_gap) scans lesson files for `Domain:\s*` tags. Four filtering-domain lessons (L-1005, L-1007, L-1008, L-1018) had no Domain tag despite being core outputs of 4 DOMEX-FLT sessions. orient.py reported "filtering (6 experiments) — no lessons extracted yet" even though 4 lessons existed.

Root cause: DOMEX session protocol has no creation-time enforcement for Domain tags (only `level=` tags are enforced via open_lane.py). Domain tags are aspirational protocol, not structural (L-601 pattern).

## Fix applied
Added `**Domain**: filtering` to L-1005, L-1007, L-1008, L-1018. Filtering INDEX.md updated with all 4 lessons + frontier resolution status.

## Rule
L-601 applies: Domain tag adoption will stay ~0% unless enforced at creation time. Target: add Domain field to lesson template, or check in lesson_collision_check.py. Aspirational protocol without gate = decays to floor.
