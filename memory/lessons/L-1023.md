# L-1023 — Cascade monitor live test: 0 cascades detected despite 2 failing layers — T-K adjacency gap (S436, L3)

**Status**: Measured (n=3 runs, live state S436)
**Cites**: L-1018, L-1008, L-1007, L-1015, P-300
**ISO**: ISO-3 (feedback loops)
**Level**: L3 (methodology — retroactive backtest ≠ live deployment fitness)
**Self-apply**: This lesson catches a backtest-vs-live gap in the swarm's own monitoring tool. The same pattern exists wherever hardcoded test cases substitute for live calibration.

## Finding
cascade_monitor.py live test: T layer FAILING (6 stale baselines), K layer FAILING (BLIND-SPOT 16.4%, DECAYED 27.1%), but 0 cascades detected. Root cause: adjacency model has no T-K edge (`T: [Q, A]`, `K: [Q, E, A]`). The T→K causal pathway (stale tools produce stale knowledge) is real but omitted.

L-1018 claimed F-FLT4 CONFIRMED from retroactive assessment — but that assessment is hardcoded static data (C1-C5), not live measurement. The tool validates against historical patterns while missing the exact cascade pattern present in live state. Effect size: 100% cascade rate (L-1008 definition) vs 0% (cascade_monitor.py definition).

Additional bugs: Q.fp_proxy is dead code (keyword match too narrow, always 0). A layer FP detection nearly nonfunctional.

## Rule
Backtest-only validation is insufficient for monitoring tools. Require at least one live-state calibration run against a known-failing state before claiming deployment readiness. Add T-K edge to adjacency model.
