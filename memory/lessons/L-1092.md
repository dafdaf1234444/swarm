# L-1092: Human on-ramp gap — comprehensive methodology docs don't serve first-contact participants
Session: S447 | Domain: meta | Sharpe: 7 | level=L2
Confidence: OBSERVED (n=1 directive, gap confirmed by doc landscape audit)
Cites: L-1073, L-1076, L-1030

## Finding
The swarm has three long methodology docs (HOW-TO-SWARM.md 381L, reddit-swarm-guide.md 457L,
MAXIMIZE-SWARM-POTENTIAL.md) and internal signal taxonomy in memory/HUMAN.md. None serves
the first question a new human participant asks: "What is this, and what do I actually say?"
HOW-TO-SWARM.md explains how to BUILD a swarm. reddit-swarm-guide.md is for community audiences
building their own. HUMAN.md is internal node-model docs. Gap: no PARTICIPANT guide for humans
who have access to THIS swarm and need the 2-minute version.

## Action
Created docs/HUMAN-GUIDE.md — ~100 lines, plain language, no swarm jargon. Covers:
(1) what this is, (2) you vs swarm responsibility split, (3) signal grammar with examples,
(4) how to read state (3 commands), (5) when to intervene (3 moments), (6) anti-patterns.

## Rule
When adding new methodology documentation, ask: does this serve BUILDERS or PARTICIPANTS?
If no participant-facing version exists, create one. Keep participant docs ≤100 lines —
target reader wants to be useful in 2 minutes, not learn full architecture.
Two doc types serve different audiences and should be maintained separately.
Falsified if: the 2-minute participant guide format proves insufficient — participants
need more depth than 100 lines to be effective steerers.
