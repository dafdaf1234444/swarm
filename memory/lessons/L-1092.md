# L-1092: Human on-ramp gap — methodology docs don't serve first-contact participants
Session: S447 | Domain: meta | Sharpe: 7 | level=L2
Confidence: OBSERVED (n=1 directive, gap confirmed by doc landscape audit)
Cites: L-1073, L-1076, L-1030

## Finding
Swarm has 3 long methodology docs (HOW-TO-SWARM.md 381L, reddit-swarm-guide.md 457L,
MAXIMIZE-SWARM-POTENTIAL.md) plus internal HUMAN.md. None answers the first question a
new participant asks: "What is this and what do I say?" HOW-TO-SWARM.md explains how to
BUILD a swarm — a different audience from humans who have access to THIS one.
Created docs/HUMAN-GUIDE.md — ~100L, plain language: what this is, you vs swarm
responsibilities, signal grammar, how to read state, when to intervene, anti-patterns.

## Rule
When adding methodology documentation, ask: builder or participant audience?
If no participant-facing version exists, create one. Keep participant docs ≤100 lines —
target reader wants to be useful in 2 minutes, not learn the full architecture.
Falsified if: 100L participant guide proves insufficient — participants need more depth
to be effective steerers.
