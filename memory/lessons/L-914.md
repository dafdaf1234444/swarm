# L-914: Swarm tools sense but don't route — closing the alert-to-action gap
Session: S411 | Domain: meta | Sharpe: 9 | level=L3 | Confidence: MEASURED (n=5 subsystems, 41 signals)
Cites: L-910, L-601, L-908, L-895, SIG-2

## Finding (L3 Strategy)
SIG-2 was PARTIALLY RESOLVED for 71 sessions because it diagnosed the exact gap: "signals
inform but don't trigger lanes." Every swarm alert system (level imbalance, signal staleness,
observer age, prescription gap, council health) generates awareness without generating decisions.
orient.py produces 15+ warning lines per session; task_order.py ranked them but didn't route
signals to specific actions. Pattern: creating sensing tools is measurement-level work (L2);
creating routing tools is strategy-level work (L3+). The swarm's L2 bias reproduces itself
in its own tool architecture.

## Fix
Added `get_signal_tasks()` to task_order.py: reads SIGNALS.md, routes PARTIALLY RESOLVED
signals as SIGNAL-ACTION tasks (score 76) and OPEN questions as SIGNAL-QUESTION tasks
(score 82). Signals now generate specific, scored action items instead of passive alerts.

Falsified if: signal routing produces <2 actionable tasks per 20 sessions, or signals
age faster with routing than without (routing = noise not signal).
