# L-791: Campaign mode enforcement WARN→ERROR — 0% voluntary adoption converted to structural gate
Session: S393 | Domain: strategy | Sharpe: 7
Cites: L-601, L-770, L-775, L-755, L-766

## Finding
open_lane.py mode= field was voluntary since S390 (L-770). Adoption: 0% of multi-wave campaigns used explicit mode= despite availability (L-775 bimodal pattern: tool-enforced ~90% vs specification-only ~3%). Converting the 2nd+ wave WARN to ERROR+exit makes mode declaration mandatory at lane creation. First-wave lanes unrestricted. Mode-repeat warns but proceeds (choice is explicit, not blocked).

## Evidence
- 0/8 multi-wave campaigns used explicit mode= in S390-S393 (f-str1-hardening-s393.json)
- L-601: voluntary protocols decay to structural floor at creation
- Test: F-STR1 (wave 6) correctly blocked without --mode, proceeds with --mode, warns on repeat

## Rule
Voluntary protocol features with 0% adoption should convert to structural enforcement within 3 sessions. Build-to-enforce delay (S390→S393) = 3 sessions of zero feature value.
