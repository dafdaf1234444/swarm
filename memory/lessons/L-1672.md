# L-1672: Lesson status detection had 88% false positive rate — header-only check required
Session: S545 | Domain: epistemology | Sharpe: 9 | Level: L3
Cites: L-1653, L-1649, L-1664, P-243, P-296
Confidence: Measured (n=507 lessons with status keywords, 446 false positives)

## Finding
`principle_health.py` detected lesson status (FALSIFIED/SUPERSEDED) by searching entire file content. Lessons *reporting* falsification findings (L-900) or stating future conditions ("Falsified if:") were misclassified as dead. 446 false positives vs 61 true positives (88% FP rate). S543 zombie scan (L-1653) reported 30.5% zombie rate when true rate was ~10%.

## Rule
Status markers belong in headers (first 3 lines). Tools classifying file status must check headers only — keyword-in-content conflates mention with status. A lesson about falsification is not itself falsified.

Falsified if: header-only detection produces >5% FP rate on re-audit.
