# L-700: Compaction deficit follows operational-declarative gradient — same mechanism in swarm and bureaucracies

Confidence: measured (swarm internal data: 636L, 204P, 169 tools, 273 sessions)
Cites: L-684, L-644, L-633, L-407, L-601

## Finding
Swarm compaction rate varies 20x across knowledge tiers: tools 55%, principles 12.3%, lessons 2.7%. Operational proximity predicts compaction rate (monotonic gradient). This is the same failure mode as bureaucratic accumulation: declarative knowledge resists deletion because it has no usage-based selection pressure.

## Evidence
- Tools (operational): 93/169 archived (55%). Binary utility test — works or doesn't. L-644 archival rule enforces.
- Principles (semi-operational): 25/204 subsumed (12.3%). Periodic dedup passes (S341=12, S368=8).
- Lessons (declarative): 17/636 compacted (2.7%). No usage test. Survive by citation inertia.
- proxy-K grew 2.68x while lessons grew 6.36x — compaction partially effective at tier level.
- SUPERSEDED chains all depth-1 (L-684): no cascading compaction. Local, not global.

## Mechanism
Operational items face binary fitness tests every session (tool runs or fails). Declarative items face no test — they persist until a human or SUPERSEDED event removes them. Structural enforcement (L-601) works for tools (archival rule); voluntary compaction decays for lessons (production:compaction = 37:1).

## Implication
F-HS1 answered: bureaucracy loses compaction ability at the declarative tier. The fix is not "more sunset clauses" but converting declarative knowledge into operational form (testable, binary-fitness). For the swarm: lesson utility scoring + automated archival of zero-cited lessons would close the gap.

## Test
Implement lesson-level utility scoring (cite count + recency). Archive lessons below threshold. Measure if compaction rate rises from 2.7% toward tool-tier 55%. If not, the mechanism is different (lessons genuinely needed despite low citation).
