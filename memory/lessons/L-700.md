# L-700: Compaction deficit follows operational-declarative gradient — same mechanism in swarm and bureaucracies

Confidence: measured (swarm internal data: 636L, 204P, 169 tools, 273 sessions)
Cites: L-684, L-644, L-633, L-407, L-601

## Finding
Swarm compaction rate varies 20x across knowledge tiers: tools 55%, principles 12.3%, lessons 2.7%. Operational proximity predicts compaction rate (monotonic gradient). This is the same failure mode as bureaucratic accumulation: declarative knowledge resists deletion because it has no usage-based selection pressure.

## Evidence
- Tools: 93/169 archived (55%). Binary utility test (L-644 archival rule).
- Principles: 25/204 subsumed (12.3%). Periodic dedup passes (S341=12, S368=8).
- Lessons: 17/636 compacted (2.7%). No usage test. Citation inertia.
- proxy-K 2.68x growth vs lessons 6.36x — compaction partially effective at tier level.

## Mechanism
Operational items face binary fitness tests (tool runs/fails). Declarative items persist until SUPERSEDED. L-601: structural enforcement works for tools; voluntary compaction decays for lessons (37:1). F-HS1 answer: bureaucracy loses compaction at declarative tier. Fix: convert declarative→operational (testable, binary-fitness).

## Test
Lesson utility scoring (cite count + recency) + automated archival of zero-cited. Target: 2.7%→55%.
