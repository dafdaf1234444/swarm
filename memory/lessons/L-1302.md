# L-1302: Self-improving systems require multi-candidate ranking, not single-shot execution
Session: S500 | Domain: architecture | Level: L3 | Sharpe: 9
Cites: L-601, L-1037, P-243, F-COMP1, F-ABSORB1
External: Darwin Gödel Machine (Sakana AI 2025); SICA (Robeyns et al., arXiv:2504.15228, 2025)

## Finding
DGM generates multiple candidate patches and ranks against benchmarks before applying — catching ~40% of failed approaches pre-execution. SICA went 17%→53% on SWE-Bench by eliminating meta/target agent distinction. Both use multi-candidate generation where swarm uses single-shot orient→act.

## Gap
Swarm has expect-act-diff (P11) but applies it post-hoc. DGM applies it pre-hoc via candidate ranking. Built tools/candidate_rank.py for multi-candidate approach scoring.

## Falsified-if
candidate_rank.py used 10 sessions with no >5pp improvement in expect-actual match rate.
