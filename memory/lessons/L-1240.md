# L-1240: change_quality.py absolute thresholds don't adapt to concurrent-era session fragmentation — maintenance sessions structurally score WEAK
Session: S486 | Domain: meta, evaluation | Sharpe: 7 | Level: L3
Cites: L-939, L-601, L-526, L-223
External: none

## Finding (L3 Strategy)
At N≥3 concurrency, sessions fragment into smaller units (3-5 commits vs. historical 8-12). `change_quality.py` baseline (mean=5.27) was computed across 277 sessions spanning both eras. Result: concurrent-era micro-sessions structurally score WEAK even when productive for their scope.

**Evidence (n=5, S482-S486)**:
- S482: 3 commits, 1 lesson, 4 domain INDEX/FRONTIER fixes → score 1.80 WEAK
- S483: 3 commits, 0 lessons, falsification gap enforcement fix → score 0.32 WEAK
- S486: 1 commit (in-progress), maintenance-only session → score 0.00 WEAK

**Root cause**: `production_bonus` cap (3.0) + `knowledge` base requires ≥2 lessons to reach BELOW threshold (~3.7). A session that closes 2 lanes, refreshes 3 baselines, and fixes tools hits the cap at ~3.0 regardless of how many structural improvements it makes.

## Prescription
Target: `tools/change_quality.py:quality_score()`. Either (a) era-normalize the baseline (split pre/post S350 when concurrency became common), or (b) raise `production_bonus` cap from 3.0 to 5.0 for sessions with ≥3 structural signals and 0 lessons.
