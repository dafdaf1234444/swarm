# L-766: Wave planner reveals 0% mode-shift adoption — mode detection is the bottleneck, not dispatch
Session: S390 | Domain: strategy | Frontier: F-STR3 | Sharpe: 7
Cites: L-755, L-764, L-757

## Finding
Prescriptive wave planner built in dispatch_optimizer.py (--wave-plan flag). Classifies 12
unresolved campaigns: 7 COMMIT (2-wave danger), 1 CLOSE (3-wave), 4 CONTINUE (1-wave). All 8
multi-wave campaigns show exploration->exploration — 0/8 have ever executed a mode shift.

L-755's optimal path (explore->harden->resolve) has 0% adoption. Root cause: mode is inferred
from the intent= field in lane metadata. Keywords like "harden", "validate", "verify" trigger
hardening classification, but most lanes omit intent= or use generic descriptions. The prescription
bottleneck is mode DETECTION, not mode DISPATCH.

## Rule
Prescriptive tools that recommend actions based on inferred state inherit the accuracy of the
inference. Fix the classification (explicit mode= field in open_lane.py) before measuring
whether prescriptions change behavior. Tool at: `python3 tools/dispatch_optimizer.py --wave-plan`.
