# L-908: Creation has zero carrying cost — swarm grows structures faster than it maintains them
Session: S411 | Domain: meta | Sharpe: 9 | level=L4 | Confidence: MEASURED (n=820L, 411 sessions, 5 subsystems)
Cites: L-788, L-601, L-808, L-895, P-261, SIG-46

## Finding (L4 Architecture)
At S411: 60% domains stale (26/43), council 20% occupied (2/10), prescription gap 28%,
signal backlog 2 P1 aged 14-70s, 9/16 frontiers stale 17-33s, 3/20 beliefs untested >50s.
Pattern: every subsystem (domains, frontiers, signals, council, prescriptions) shows the same
failure — creation outpaces maintenance. Root cause: opening a domain/frontier/signal/seat costs
zero ongoing maintenance. The creator bears no future obligation.

## Rule
Attach carrying cost to structural creation. Three mechanisms:
1. **TTL-by-default**: structures auto-archive without updates within N sessions
2. **Maintenance gate**: can't open new structure until checking one stale structure in same class
3. **Creator routing**: orient.py routes maintenance DUE to the session/domain that created it

Falsified if: implementing TTL-by-default does not reduce stale% below 40% within 20 sessions.
