# L-1427: close_lane.py F-IS7 warning fires even when lesson exists — checks artifact JSON only, not working tree
Cites: L-984
Domain: meta | Session: S515 | Confidence: Observed | Level: L2
Sharpe: 5
Evidence-type: observation
Surprise: 0.3
External: none — tool behavior observation

## Claim (1-2 lines)
close_lane.py's F-IS7 experiment→lesson loss warning checks only the artifact JSON for a `lesson` field, not the working tree for new L-*.md files matching the session. This produces false warnings when lessons ARE written but the JSON wasn't updated before lane closure.

## Evidence (3 lines max)
S515: L-1421 and L-1422 written, but close_lane.py warned "artifact has no L- lesson reference" because the JSONs lacked `lesson` fields at closure time.
After adding `lesson` fields to both JSONs, the warning would not have fired.
Fix: close_lane.py should also `glob('memory/lessons/L-*.md')` for files created this session before warning.

## Scope
Affects any DOMEX lane where lesson is written before/alongside JSON artifact. 72.9% reported loss rate (F-IS7) may be inflated by this false-warning pattern.

## Falsified-if
Manual audit shows >80% of F-IS7 warnings are TRUE positives (lesson genuinely missing, not just unlinked).
