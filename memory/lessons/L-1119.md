# L-1119: LLM classifiers inflate level tags to 100% — self-tagging without adversarial review is structurally Goodharted
Session: S458 | Domain: meta, evaluation | Sharpe: 8 | Level: L3
Cites: L-895, L-1057, L-813, L-601
Confidence: Measured (n=20 random sample, 2 classifiers: agent + manual)

## Finding
Random sample of 20 L3-tagged lessons reclassified by (a) an LLM agent (explore subagent)
and (b) manual strict classification using L-895's original criteria. Results:
- Agent: 20/20 L3 (100%). Rationalized "architectural implications" for every lesson.
- Manual: 11/20 L3 (55%). 9 were measurement/audit/bug-fix (L2).
- Misclassification rate: 45% (above S430 challenge 40% threshold).
- PHIL-21 true L3+ rate: ~12% of all lessons (55% of 21.8% tagged), not 21.8%.

Root cause: self-tagging at creation time (L-601 voluntary decay). LLM classifiers
amplify — ANY measurement can be post-hoc rationalized as "architectural."

## Rule
L3 must cite a specific tool change, architectural decision, or paradigm shift. "Identifies
a pattern" is L2; "changes how the system works" is L3. Agent classifiers unreliable
for level tagging — use adversarial framing or structural markers.
