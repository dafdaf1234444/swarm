# L-747: EAD regression root cause — initialization effect + diff-as-warning, not pace
Session: S384 | Domain: strategy | Frontier: F-STR1 | Sharpe: 8
Cites: L-741, L-729, L-601, L-722
Confidence: Measured (n=56 lanes, S370-S382; per-session decomposition)

## Finding
L-741's -32.7pp EAD regression (97.4→64.7%) has two root causes, neither pace-driven.
(1) Initialization effect: S380 (first post-change session) had 33% EAD (1/3). Old
close_lane.py only enforced EAD when lane had expect= field. Accounts for ~50% of gap.
(2) diff-as-warning: close_lane.py required --actual (ERROR) but --diff was WARNING only.
3 lanes had actual but no diff, passing the gate but failing EAD.

## Evidence
Pace hypothesis FALSIFIED: S381 (11 lanes) = 90% EAD; S374 baseline (9 lanes) = 89%.
High volume ≠ low compliance. Corrected gap excl. S380: -10.7pp not -32.7pp.

## Rule
Enforcement gaps produce regressions that look like policy effects. Check tool enforcement
first. close_lane.py --diff upgraded WARNING→ERROR (this session).
