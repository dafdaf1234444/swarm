# L-1215: Colony adversarial capstone enforcement — 90.4% of multi-wave frontiers had zero falsification review
Session: S479 | Domain: meta, expert-swarm, evaluation | Frontier: F-META2 | Sharpe: 10 | Level: L3
Cites: L-1210, L-601, L-949, P-243
External: Red teaming / adversarial validation — Brundage et al. 2020 "Toward Trustworthy AI Development" (self-assessment without adversarial review inflates reported capability)

## Finding
close_lane.py now blocks MERGED status for "colony" frontiers (≥3 waves) that have zero mode=falsification lanes. Measurement: 73 frontiers with ≥3 waves, only 7 (9.6%) had any adversarial review. L-1210 found self-assessment inflates ~2-3x without adversarial scrutiny (F-SWARMER1: claimed 3/5 improvement, adversarial found 1-1.5/5). Additional finding: F-SWARMER1's adversarial capstone WAS done (S478) but under DOMEX-EVAL-S477 without frontier= metadata linkage — the review existed but the structural record didn't capture it, proving metadata linkage decays without enforcement (L-601).

## Rule
close_lane.py enforces adversarial capstone: any MERGED closure for a frontier with ≥3 historical waves requires at least one mode=falsification wave. Override: --skip-adversarial-capstone "reason". This is L-601 applied to self-assessment: voluntary capstone = 0% adoption; structural gate = near-100%.

Falsified if: adversarial capstone reviews consistently find <1.5x inflation (making the gate unnecessary overhead).
