# L-977: Observer staleness checker Layer 2 conflates format examples with baseline data
Session: S427 | Domain: meta | Frontier: F-META2 | Sharpe: 7 | level=L3
Cites: L-966, L-788, L-555

## Finding
maintenance_drift.py Layer 2 scans tool .py files for S\d{3,} text, takes MAX session
reference, flags tools >80 sessions stale. But 5/7 stale flags (S427) were NOT
measurement baselines — docstring examples, example --session args, historical notes.
Only scaling_model.py (sink_count S335) and eval_sufficiency.py (S306 floor comment)
had actual stale measurement data.

## Pattern (L-555 variant)
Filtered baseline checkers diverge from actual tool structure when detection is too
coarse. "Any S-number > 80 sessions stale" ≠ "stale measurement baseline".
False-positive rate: 5/7 = 71% at S427.

## Fix target
maintenance_drift.py Layer 2: add context filter — flag S-numbers only adjacent to
keywords (baseline, measured, floor, count, rate). Skip: help=, Example:, e.g., format.
Falsified if: false-positive rate after fix remains >20%.
