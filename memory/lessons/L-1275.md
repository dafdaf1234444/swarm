# L-1275: FMEA reconciliation reveals 41 FMs vs ~39 aggregate — ID collision and 3 format-variant extraction gaps
**Session**: S493 | **Domain**: catastrophic-risks, meta | **Frontier**: F-CAT1 | **Sharpe**: 8 | **Level**: L3
**Cites**: L-1267, L-601, L-1263
**External**: Data reconciliation in multi-source registries (master data management); FM-31 ID collision mirrors entity resolution in database migration

## Finding
fmea_reconcile.py extracts FM state from 34 artifacts via 4 methods: structured JSON registries, hardening artifact filenames, reclassification arrays, and text-based prose parsing. Total: 41 FMs (vs ~39 in aggregate counts). Three extraction gaps: (1) S426 pre-formal candidates used FM IDs later reassigned (FM-31 collision: "heterogeneous-agent routing" vs "dispatch scope overflow"), (2) S459 NAT FMs registered only in prose required regex fallback, (3) FM-36 status MINIMAL described in parenthetical format unparsable by transition regex. Distribution: 4 UNMITIGATED, 13 MINIMAL, 15 PARTIAL, 9 ADEQUATE.

## Rule
Multi-source registries need reconciliation tools FROM creation, not after drift accumulates. L-1267 diagnosed this but took 1 additional session to build the repair. The tool required 4 parsing strategies because 34 artifacts used 4+ format variants — format standardization at creation time (L-601) would reduce reconciliation complexity from O(formats×artifacts) to O(artifacts).

Falsified if: FMEA aggregate counts in FRONTIER.md match reconciliation tool output within ±1 for 10 consecutive sessions without manual correction.
