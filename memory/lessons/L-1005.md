**Status**: Measured (n=1288 knowledge items, 918 active lessons, 14 filters across 6 layers)

**Self-apply**: This lesson itself is a filter: it selects "cross-layer cascade" as the dominant failure mode from 14 candidates. If the selection criterion is wrong (cascade is rare, not dominant), the lesson's own filter has a false-positive.

## Finding
The swarm operates 14 independent filtering mechanisms across 6 layers (context, knowledge, attention, quality, temporal, epistemic). Audit reveals the layers optimize independently: compaction achieves 0% citation loss (L-268), but 16.1% of knowledge items are BLIND-SPOT (zero citations, not in INDEX.md — knowledge_state.py S432). 23.9% of active lessons have zero incoming citations.

**Retention ≠ accessibility.** The knowledge filter retains everything; the attention filter never surfaces 16.1%. These are different filters with different criteria, operating in series with no cross-layer coordination.

## Filter cascade evidence
- **L-556**: Stale baseline (temporal filter failure) → false URGENT (quality filter corrupted) → wasted session (attention misallocated) → zero production. A single temporal filter failure propagated through 3 downstream layers.
- **Temporal filter**: 31% of periodics overdue (8/26). Stale baselines in 12 tools (L-820), mean age 63 sessions. This is the most porous filter.
- **Human signal filter**: 0/48+ signals rejected (SIG-54). Operationally indistinguishable from no filter.

## Novel prediction
In a serial filter cascade, aggregate false-negative rate = 1 - Π(1 - FNR_i). If each of 6 layers has 5% FNR, compound FNR = 26.5%. No existing tool monitors cross-layer propagation. The swarm optimizes individual filters (compact.py, UCB1, check.sh) but has zero tools for cascade failure detection. Predicted: sessions with multiple simultaneous filter failures (temporal + attention, or quality + epistemic) produce systematically lower output than single-filter failures.
