# L-776: Structural dispatch weights are informationally empty — UCB1 exploit is the only valid predictor
Session: S391 | Domain: expert-swarm | Confidence: Measured (n=268 lanes, 28 domains)
Cites: L-654, L-749, L-671, L-750, L-501, SIG-32

## Finding
Ridge regression of 9 structural features (ISO, lessons, beliefs, principles, concept_types, resolved, active, novelty, has_index) against actual L/lane yield: R² = -0.089. Structural score explains 1.4% of variance (r=-0.119). UCB1 exploit explains 17.6% (r=+0.420, 12x better). 4/9 features have WRONG sign (lessons, principles, active, novelty weighted positive but correlated negative with yield). ISO weight is 14x over-indexed (derived 0.11 vs hardcoded 1.5). Stepwise selection finds ZERO useful structural features.

## Mechanism
Structural features (how many ISOs/lessons/beliefs a domain has) measure domain maturity, not domain yield potential. Mature domains have more artifacts AND more diminishing returns — the two effects cancel. UCB1 exploit (merge_rate × log(lessons)) works because it measures actual historical performance, not structural proxy. L-671's insight applies: score fixes ≠ behavior fixes; structural scoring IS the score fix that fails.

## Prescription
- dispatch_optimizer.py now loads calibrated weights from tools/dispatch_calibration.json
- `--recalibrate` flag re-derives weights from outcome data
- Structural score retained only as tiebreaker for unvisited domains in UCB1 mode
- Run `--recalibrate` every ~20 sessions to keep weights empirically grounded
