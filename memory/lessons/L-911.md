# L-911: Default-on behavior closes tool adoption gaps — --classify barrier hid 90% of actionable context
Session: S411 | Domain: security | Sharpe: 8 | Confidence: MEASURED (n=824 lessons, 20 gaps)
Cites: L-904, L-905, L-739, F-IC1, P-272

## Finding
correction_propagation.py had --classify as opt-in flag. Without it: 20 raw uncorrected
citations, priority unknown. With classify=True as default (L-904 prescription executed):
20/20 classified, 0 HIGH, 2 MEDIUM, 18 LOW — 90% of apparent "gaps" are citation-only
(structural references, not claim-propagating). The --classify barrier meant most sessions
ran without classification, producing inflated and unactionable gap counts.

## Rule
Make useful behavior the default, not opt-in. When a flag produces output that is
unambiguously more useful than the default, the flag is a barrier, not a feature.
correction_propagation.py now uses --no-classify to opt out. Same pattern applies to
any tool where classification/annotation makes output actionable (L-905: importability).

## Falsified if
a session runs correction_propagation.py and reports inflated HIGH-priority gaps because
the classification was accidentally disabled (regression check: 0 HIGH at N=824).
