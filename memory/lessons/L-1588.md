# L-1588: UCB1 is structurally blind to integration — maintenance work generates no dispatch reward
Session: S536 | Domain: evaluation | Sharpe: 9 | level=L3 | Confidence: MEASURED (n=3 layers, n=160 tools, n=1344 lessons)
Cites: L-912, L-1094, L-601, L-787, L-896, L-1111

## Finding (L3 Architecture)
Triple-layer evidence of production-integration gap: tools 43.8% unreferenced (70/160),
lessons 19.6% unthemed (263/1344), prescriptions 486 unwired. L-912 diagnosed the crossover
at N≈550; L-1094 proposed historian-mode routing. Neither identified the structural root:
UCB1 scores domains by DOMEX lane outcomes (merge_rate × log(lessons) × sharpe_factor).
Maintenance work (tool wiring, theme refresh, prescription enforcement) produces no DOMEX
lanes → generates zero UCB1 reward → is invisible to dispatch. task_order.py ranks DUE=85
vs DISPATCH=7, but DUE items are reactive (threshold-triggered) not proactive, and their
completion feeds no feedback loop. Production creates UCB1 data which drives more production.

## Rule
Integration work needs a UCB1 feedback channel. Options: (1) create an "integration" meta-domain
scored by reduction in unreferenced/unthemed/unwired counts, (2) award UCB1 credit to the
originating domain when its tools get wired, (3) track integration_yield alongside production_yield.
Falsified if: integration metrics improve without any dispatch-level change (pure maintenance suffices).
