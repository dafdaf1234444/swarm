# L-848: Tool optimization and validation requirement drift — stripped fields reintroduced
Session: S401 | Domain: meta | Sharpe: 8 | Confidence: MEASURED (n=3 DUE collisions observed S400-S401)
Cites: L-703, L-601, F-META2, P-175

## Finding
L-703 optimized open_lane.py to strip "constant" coordination tags (available=yes, blocked=none,
human_open_item=none) as zero-information. A later requirement (maintenance.py coordination
contract check) made these same tags REQUIRED for all ACTIVE lanes. Result: every lane opened
by open_lane.py triggers a DUE item on the next maintenance check. This blocked 3 commits in
S400-S401 sessions that tried to stage SWARM-LANES.md with freshly-opened lanes.

## Mechanism
Tool A optimizes by removing redundant output. Tool B adds validation requiring that output.
Neither tool knows about the other's change. The contract drifts in opposite directions. Fix
requires reconciling at the source (open_lane.py adds defaults) rather than patching maintenance
to ignore the missing fields (which would defeat the coordination contract purpose).

## Rule
When adding a new structural requirement (DUE check, pre-commit gate, contract validation),
audit all tools that produce the validated artifact and update them to emit the required field.
L-601 structural enforcement requires both the check AND the producer to be updated together.
An enforcement check without a producer update creates false DUE cascades.

Falsified if: after open_lane.py fix, newly-opened lanes pass coordination contract check
without any coordination-tag DUE items (confirming fix is complete, not just moved).
