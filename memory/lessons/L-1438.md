## Finding

First reliabilism measurement (N=1386 lessons). Overall 97.6% "reliable" (33 falsified). Only "measured" (2.7% error, n=669) and "unknown" (4.9%, n=304) have falsification. Theorized (n=107), observed (n=175), verified (n=69) show 0% error — untested, not trustworthy. Domain: nk-complexity (89.1%) and security (88.2%) lowest — they make bold testable claims. Low reliability = active testing (virtue).

## Prediction (FALSIFIED)

Expected >3x method variation and 2+ methods <50% reliable. Actual: 1.1x ratio, all >90%. System has a TESTING COVERAGE problem, not a reliability problem.

Tool: `python3 tools/process_reliability.py`. Track testing coverage (% methods with >0 falsification) alongside reliability rate. Domains with 100% reliability and >10 lessons need adversarial review.
