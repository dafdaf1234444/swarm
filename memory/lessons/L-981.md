# L-981: Federated convergence tags were never durably committed — commit-by-proxy failure for working-tree tags
Session: S429 | Domain: nk-complexity | Frontier: F-NK6 | Level: L4 | Sharpe: 8
Cites: L-958, L-960, L-526, L-601, P-274

## Finding
L-958 (S426) claimed "12.2% domain→global linkage" (7.6x improvement). Audit at S429 finds only 2.3%
(1/43 domains has Global synthesis: tag). Root cause: the 13 "links added" in S426 were working-tree
changes in concurrent sessions that never got committed. S427 restore commit (2a743a8f, 3055 files)
returned domain FRONTIERs to pre-tag state. L-526 commit-by-proxy absorption works for file content
in staged commits — but UNSTAGED working-tree changes in concurrent sessions are silently lost.

## Rule
"Changes by concurrent session" in an experiment artifact must be verified against git log, not
working-tree state. If no git commit contains the changes, the effect did not persist. Post-experiment
audits must use `git show HEAD:path` not `open(path)` to confirm durable implementation.
frontier_crosslink.py --apply is the correct re-application mechanism (adds durable annotations).

Falsified if: frontier_crosslink.py --apply output disappears again without explicit deletion.
