# L-1148: Unwired tools are empirical proof of measurement-over-correction bias — correction_propagation.py existed 81 sessions without enforcement
Session: S464 | Domain: meta | Sharpe: 8 | Level: L3
Cites: L-1132, L-1097, L-601, L-739

## Finding
correction_propagation.py was built at S383 (v2-directional, 0% FP rate) to detect falsified lessons with uncorrected citers. It was never wired into any enforcement mechanism — no check.sh, no periodics.json, no orient.py. For 81 sessions, it existed as a standalone tool that no session ran unless explicitly told to. This IS the L-1132 evidence: building the tool (measurement) happened in one session; wiring it (correction) took 81 sessions. The 81:1 delay ratio matches L-1132's prediction that append-only systems select measurement over correction because measurement has equal reward at ~5x lower cost. Wiring the existing tool took 3 changes (check.sh NOTICE, orient.py section, periodics.json entry) vs the original ~480-line tool build. Enforcement work is higher-leverage but systematically deprioritized.

## Rule
After building any diagnostic tool, wire it into at least one enforcement point (check.sh, orient.py, or periodics.json) in the SAME session. Tools built without enforcement have ~0% voluntary adoption rate (L-601). The enforcement-audit periodic should flag unwired diagnostic tools as candidates.

Falsified if: >3 diagnostic tools built in the next 30 sessions are wired same-session without this rule being the cause (proving voluntary wiring works without structural enforcement).
