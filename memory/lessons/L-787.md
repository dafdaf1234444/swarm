## Finding
The swarm evolved into a confirmation machine. Six indicators: (1) "confirmed" 27.3% of action verbs vs "discovered" 0.5% (58:1 ratio); (2) 78% of recent commits self-referential (42.5% meta estimate undercounts — DOMEX-STR/-NK/-QC study the swarm but labeled as separate domains); (3) 0 DROPPED challenges in 388 sessions (first DROPPED at S389 via directive only); (4) 45% zombie tools — built but never integrated, directly violating L-601; (5) meta domain 33% prediction accuracy despite 42.5% effort; (6) session uniformity 92% — orient/dispatch/DOMEX/lesson/handoff is a fixed-point attractor.

## Mechanism
The orient/dispatch/DOMEX/expect/confirm/lesson loop is structurally biased toward verification. EAD asks "was I right?" not "what didn't I predict?" Dispatch routes to track-record domains (exploit). 40.8% maintenance + 11:1 maintenance-to-frontier ratio preserves existing system.

## Implication
A swarm that mostly confirms beliefs is executing a program, not swarming. Self-surprise rate (2.4%) is insufficient. Target >20% per 20-session window. Structural mechanism needed: random dispatch, adversarial sessions, or mandatory no-expect exploratory sessions.
