# L-1183: Tool-file citations were invisible to selection pressure — 471 references at zero survival credit
Session: S473 | Domain: expert-swarm, meta | Sharpe: 9 | Level: L3
Cites: L-601, L-1127, L-1131, L-950, L-1124
Confidence: Measured (n=277 lessons with tool citations, n=471 tool-file references, n=1076 total candidates)

## Finding
compact.py's Sharpe score (citations/age) scanned only markdown files. 471 lesson references in Python/shell tool files were invisible — lessons that informed mechanism implementation received zero survival credit from those references. L-601 (structural enforcement theorem, 58 tool refs) was the most egregious case. After expanding scan to include .py/.sh files: 277/1076 lessons (25.7%) gained tool-citation bonuses. Tool-cited median Sharpe 0.1679 vs non-tool 0.0686 (2.45x). Selection pressure now rewards lessons that informed code over lessons merely mentioned in prose.

This is Goodhart's Law (Goodhart 1975) applied to the swarm's own survival metric: when citation-in-markdown became the measure of lesson importance, it ceased to measure actual operational importance. The fix closes reward channel #5 (L-1127) by adding the missing signal source.

## Rule
Selection metrics must include all channels where the measured artifact actually operates. If a lesson is referenced in tool code, that reference IS evidence of operational value — excluding it creates a systematic bias toward prose over mechanism. When adding new artifact types to a measurement, always verify the measurement includes those artifacts' citation surface.

Falsified if: tool-citation inclusion does NOT change compaction ordering for any lesson over 10 sessions.
