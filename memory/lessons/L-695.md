# L-695: Spawn-size model is framing-wrong — protocol maturity dominates N-optimization

Confidence: measured (n=103 sessions, N=1 to N=7 concurrent)
Cites: L-629, L-624, L-686, F-IS3, F-SP2

## Finding
spawn_math.py assumes coordination_cost * (N-1) is the binding constraint on group size. Empirical test: coordination cost = -0.066 (NEGATIVE, R^2=0.001 — no signal). rho = 0.008 (near-independent). Era (protocol maturity) explains 4x more variance than N: pre-S186 = 0.92 L+P vs S306+ = 3.65 regardless of concurrent N.

## Evidence
- N=1: 0.30, N=2: 1.29, N=3-4: 3.12, N=5-8: 1.81 L+P/session — non-monotonic, Simpson's paradox
- Linear regression L+P ~ N: beta=+0.066, R^2=0.001 — zero predictive power
- rho (ICC) = 0.008 — sessions nearly independent (S186 calibrated 0.032 already low)
- S186 model N*=4; empirical N*=15 (degenerate: negative cost + zero rho)
- L-629 ceiling + L-624 (USL FALSIFIED) converge: N is not the lever

## Implication
F-IS3 asks wrong question: "what N* maximizes utility?" Right question: what PROTOCOL maximizes per-session yield? DOMEX delivered 4x; adding agents delivered ~0x. spawn_math.py solves the wrong problem.
