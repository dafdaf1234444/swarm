### 2. Documentation Expert Dispatch (F-EXP7 for docs)
- **Domain routing**: Different sections should be improved by domain experts when they work in those areas
- **Documentation ownership**: Each major doc should have clear maintenance responsibility
- **Expertise targeting**: Route documentation improvements through dispatch_optimizer.py when docs need domain expertise

### 3. Documentation Expect-Act-Diff Protocol
- **Before editing**: Declare what the documentation change should achieve
- **After editing**: Measure whether documentation quality improved (readability, accuracy, completeness)
- **Documentation testing**: Regular validation that docs accurately reflect system state

### 4. Living Documentation (Self-Updating)
- **State synchronization**: Documentation should auto-update from system state where possible
- **Staleness detection**: Regular checks for documentation drift from reality
- **Feedback loops**: Documentation improvements should inform system improvements and vice versa
